# Documentation

At GrowthX, documentation is foundational to how we work. As a fully remote, async, international team, our ability to thrive depends on **clear, well-organized knowledge** that's accessible to everyone.

## Why documentation matters

**Efficiency** — Solve problems and share updates without waiting for a meeting or real-time response.

**Freedom** — Team members work across time zones. Good docs eliminate blockers even when someone's offline.

**Scalability** — This is how a remote org functions without the constraints of a physical office or fixed hours.

## What a documentation culture means

We collectively work to:
- Keep our knowledge base **useful, tidy, and up to date**
- Fix errors whenever they arise
- Add missing information as soon as we spot the gap
- Enable others to learn and build on our work

Documentation is our **shared brain** — it eliminates confusion, accelerates progress, and lets us work effectively across the globe.

## What counts as documentation

Anything written down that others can read, learn from, and expand on. It could be a detailed Notion page explaining a workflow, notes from a meeting, or a quick reference guide. The point is: **write it down** in a way others can understand and use.

## When to document

Document whenever:
- You're working on something others might reference later
- A process or decision has been made that could impact others
- You've learned something useful that others can benefit from

The goal: **capture knowledge** that helps others understand what you did, why you did it, and how they can build on it.

## Single source of truth

We work with various tools — Google Drive, Slack, Notion — but we designate **one place** as the single source of truth (SSoT). At GrowthX, that's **Notion**.

Why Notion:
- **Availability**: Everyone has access, and it's the most current repository
- **Quality**: Revision control keeps data accurate and easy to follow
- **Extensibility**: Data can be connected or exported to other systems
- **Built-in AI**: Helps everyone create, edit, and improve documentation

> **Note:** While we use other tools for specific needs, **Notion is the primary hub for documentation**. Don't scatter information across platforms.

## Who owns documentation

- **Page owners**: Every Notion page has an assigned owner responsible for keeping it accurate and current
- **Departments**: Documentation is ultimately owned by the department or subject matter experts where it originates
- **Everyone contributes**: If you notice something that needs improvement but can't edit the page, let the owner know

> **Tip:** Before creating new documentation, check if a page on the topic already exists. If it does, update it. If not, create a new one that meets our standards for clarity and accessibility.

## Best practices

- **Be clear and concise**: Avoid jargon. Provide context for complex concepts.
- **Add value**: Only document what helps others move forward. Don't create noise.
- **Keep it updated**: Stale information misleads. Update outdated docs or inform the owner.
- **Organize wisely**: Structure information logically. Link related documents.
- **Include dates**: Add timestamps or revision notes so everyone knows how current the info is.

## Benefits

- **Reduces dependency**: Team members find answers without waiting for someone else
- **Accelerates onboarding**: New hires get up to speed faster through detailed docs
- **Enhances collaboration**: A shared knowledge base reduces misunderstandings and keeps everyone aligned
