# How We Use Slack

Slack is our office. Communication is fundamental to a successful remote culture, so we take how we use Slack seriously.

> **Note:** **We keep the number of channels to a minimum.** If you're creating a channel, follow the patterns described here.

## Slack vs. Linear

Think about urgency and expectations when choosing between these tools:

| Urgency | Tool | Use for |
|---------|------|---------|
| **Urgent** | DMs or phone | When you need an answer immediately |
| **Important** | Linear ticket | Real requests — optionally follow up with a Slack @mention |
| **Non-important** | Slack channels | Back-and-forth, idea exchanges. Create a Linear ticket if a real need emerges |

## Avoid DMs

DMs should only be used for urgent communication. They reduce visibility and transparency, and disrupt people with notifications. Use the appropriate channel instead — we have general channels, functional channels, and project channels available.

## Channel naming conventions

Our channels are organized by area:

| Prefix | Area | Example |
|--------|------|---------|
| `d-` | Delivery (client-facing teams) | `d-general`, `d-directors` |
| `b-` | Build (sales, ops, EPD) | `b-epd`, `b-ops-general` |
| `all-` | Everyone | `all-general`, `all-announcements` |
| `d-ext-` | External client channels | `d-ext-client-name` |
| `d-int-` | Internal client discussion | `d-int-client-name` |
| `ext-` | Partners, sales leads | `ext-partner-name` |
| `-proj-` | Temporary project work | `b-proj-website-redesign` |
| `-alert` | Automations | `b-epd-alerts` |

## Access for contractors

- Freelancers and external contractors may only be added as **Slack Guests** (single or multi-channel)
- External contractors can only join internal channels. To join external/shared client channels, they need a GrowthX email address
- Request access through **#b-it-and-tools**

> **Note:** This doesn't apply to employees or contractors added via Deel, who receive GrowthX email addresses and full Slack membership.

## Working in the open

We share progress and learnings regularly through daily Geekbot check-ins. These brief updates reduce meetings, increase transparency, and help teams stay aligned.

> **Warning:** Consistent participation in daily check-ins is required. Missing updates for more than a day will prompt follow-up from your manager.

## Organizing your sidebar

You can organize your Slack sidebar however you want, but consider:
- Priority channels at the top
- Client channels grouped together
- Build / Delivery separation
- Muted channels for reference
