# Design and development

## TL;DR

- Design and development is a shared resource for accounts that need web or visual support.
- Handles website changes, design work, technical implementation, and special client initiatives.
- Not every account needs this capability — deployed based on engagement scope.
- Works closely with EMs, editors, and forward-deployed engineers on client-facing deliverables.

## What this role does

Design and development provides web and visual support that goes beyond content production. When an account needs landing pages built, website sections redesigned, visual assets created, or technical implementations that editors can't handle — that's where design/dev comes in.

You're a [shared resource](/company/organization#the-two-operating-models) deployed across accounts based on need. The engagement manager identifies when web or design support is required and pulls you in.

## Key responsibilities

- **Web development** — Build and modify client websites, landing pages, content hubs, and conversion-focused pages.
- **Design work** — Create visual assets, page layouts, infographics, and branded materials that support content strategy.
- **Technical implementation** — Handle CMS customization, tracking setup, structured data implementation, and integrations.
- **Special initiatives** — Support one-off projects like site redesigns, new section launches, or interactive content.

## Accountabilities

| What you own | How it's measured |
|-------------|-------------------|
| Deliverable quality | Visual quality, technical soundness, brand consistency |
| Turnaround time | Speed of delivery relative to project scope |
| Implementation accuracy | Technical correctness, cross-browser compatibility, performance |

## What great looks like

- Deliverables look professional and align with the client's brand standards
- Technical work is clean, performant, and doesn't introduce issues
- You understand the strategic purpose behind each request — not just what to build, but why
- Projects are scoped accurately and delivered on time
- You proactively flag when a request could be done better or differently

## How deployment works

- **Project-based** — Most work is scoped as specific projects with clear deliverables and timelines
- **On-demand support** — Quick-turnaround requests for smaller changes or fixes
- **Strategic initiatives** — Larger engagements for site redesigns, new content hubs, or conversion optimization

## Key relationships

| Who | How you work together |
|-----|----------------------|
| Engagement managers | They identify the need and define the brief. You execute. |
| Editors | Content and design often work together — content informs layout, design enhances content. |
| Forward-deployed engineers | Coordinate on technical implementations that span both AI workflows and web. |
| SEO specialists | Ensure web work supports SEO goals — site speed, structure, crawlability. |
