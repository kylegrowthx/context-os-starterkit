# Forward-deployed engineer

## TL;DR

- Forward-deployed engineers (FDEs) build and customize AI workflows for specific client needs.
- Embedded with delivery pods as technical co-pilots — one engineer per pod.
- Handle workspace setup, AI workflow customization, pipeline configuration, and technical implementation.
- Shared resource — deployed where needed, not permanently assigned to every account.

## What this role does

You're the bridge between GrowthX's AI platform and the client's specific needs. Every client has unique brand voice, content requirements, and workflows. You take the platform's capabilities and configure them to produce great results for each engagement.

You embed directly with [delivery pods](/delivery/teams-and-operations#pods-growth-execution), working alongside engagement managers and editors. When they need a workflow customized, a pipeline optimized, or a technical problem solved — you're the person.

## Key responsibilities

- **Workflow customization** — Build and configure AI workflows for client-specific requirements. Adjust prompts, calibrate outputs, optimize pipelines.
- **Workspace setup** — Configure client workspaces in ContentOS. Set up integrations, import context, establish the technical foundation for the engagement.
- **Pipeline optimization** — Continuously improve workflow performance based on production data. Reduce edit rates, increase zero-edit outputs, improve output quality.
- **Rapid response** — Quick turnaround on client support and troubleshooting. Surface and resolve production issues before they impact delivery.
- **Technical documentation** — Document patterns, configurations, and solutions so knowledge compounds across the team.
- **Reusable components** — Build components that accelerate future client implementations. Every new pattern should make the next implementation faster.

## Accountabilities

| What you own | How it's measured |
|-------------|-------------------|
| Workflow quality | Reduction in average edits per document |
| Automation rate | Percentage of zero-edit outputs |
| Implementation speed | Time from engagement start to fully configured workspace |
| Support volume | Reduction in production issues and support requests |

## What great looks like

- AI workflows produce outputs that editors barely need to touch
- New client workspaces are configured quickly and correctly
- Production issues are surfaced and resolved before editors or clients notice
- Patterns you document become the standard for future implementations
- The team's overall velocity increases because of components and patterns you've built
- You understand the delivery team's needs — not just the technical requirements, but the business context

## A typical week

- **Monday:** Review Linear board, check production metrics across pods, prioritize the week
- **Tuesday–Thursday:** Build and customize workflows, troubleshoot issues, meet with EMs and editors on pod needs
- **Friday:** Document patterns, build reusable components, contribute to team knowledge base

## Tools you'll use

- **Output framework** — The internal AI workflow engine
- **ContentOS** — Content operations platform you configure for clients
- **GitHub** — Code management, workflow code, configuration
- **Linear** — Task and project tracking
- **Cursor / Claude Code** — AI-assisted development tools

## Key relationships

| Who | How you work together |
|-----|----------------------|
| Engagement managers | They define client needs. You translate those into technical solutions. |
| Editors | They use the workflows you build. Their feedback drives your improvements. |
| Client ops team lead | Your team lead. Coaching, special projects, technical escalation. |
| Product/project manager | Prioritization, cycle planning, client pipeline coordination. |
| Senior engineers | Code reviews, complex technical projects, architectural decisions. |
| Product lane engineers | Coordinate on platform features that affect workflow capabilities. |

## Career path

- **Senior FDE (L2)** — Take on more complex implementations, lead technical projects, mentor junior engineers
- **Client ops team lead** — Transition to leadership — recruiting, coaching, and team coordination
- **Product engineering** — Move to the product lane, building ContentOS or CheckThat features
- **AI infrastructure** — Contribute to the Output framework and core AI capabilities
