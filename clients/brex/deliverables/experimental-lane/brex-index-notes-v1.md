## THE CORE CONCEPT: Indexes as Enriched Databases

**What Marcel Said:** *"It's an index of indexes. Each one of them is like a data set, an enriched data set with a description and some narrative and text around it. We can do the fuck out of that with our deep researcher. That has infinite potential."*

## HOW IT ALL CONNECTS: Hub & Spoke Architecture

### **The Web Strategy**
Marcel: *"You can create industry hubs in certain hubs as well. Those hubs then pull together content from the journal, content from the spending trends, the tools, the guides, the assets, the everything into one page dynamically."*

### **3-Layer Architecture:**

#### **Layer 1: Index of Indexes (Top Level)**
- Main directories page: `/directories` or `/indexes`
- Lists all available indexes:
  - Company Directory
  - People Directory (CFOs, CEOs, Founders)
  - Small Business Directory
  - Salary Guides
  - Vertical Software Rankings

#### **Layer 2: Individual Indexes**
- Each index is an **enriched data set table**
- Sortable, filterable grids
- Each filter combination = unique page
- Example: `/cfo-directory`
  - Filter by: Industry, company size, location, tenure
  - Each filtered view = new indexable page

#### **Layer 3: Individual Entity Pages**
- Each item in a table gets its own page
- Example: `/cfo-directory/jane-smith-cfo-acme-corp`
- Contains:
  - Description and narrative
  - Enriched data points
  - Links to related content (Spend Trends articles, tools, guides)
  - Cross-links to other directories

### **Cross-Linking Everything**
Marcel: *"I would find a way to bring all this content together more as well and cross link all of it more... The more you make that web as tightly and as densely connected as possible in a relevant way, then the higher chances you have to really do - for the whole thing to do better."*

**Example Cross-Link Flow:**
1. User searches "Who is CFO of Capital One"
2. Lands on Andrew Young's CFO profile page
3. Page includes:
   - Link to "CFO Salary Guide"
   - Link to "Capital One" company profile
   - Link to Spend Trends article: "How Capital One manages expenses"
   - Link to Tool: "CFO Budget Calculator"
   - Link to Template: "CFO Financial Planning Template"
4. Each click keeps them in Brex ecosystem

## TECHNICAL IMPLEMENTATION

### **How It Works (per Marcel):**
1. **Enriched Tables:** Not just lists - tables with multiple data points
2. **Dynamic Filtering:** Each filter state = unique URL/page
3. **Auto-Generated Pages:** Script creates individual entity pages from database
4. **Constantly Updated:** "Lists that are constantly being updated with a bunch of signals"
5. **Lead Magnets:** Make datasets downloadable for lead capture

### **Data Sources:**
- Public company data
- Funding databases (Crunchbase, PitchBook)
- Local business directories (Yelp, Google Places)
- Salary data (Glassdoor, Bureau of Labor Statistics)
- LinkedIn data for people profiles
- Deep Researcher can enrich all of this programmatically
