# Brex Web Experience: Problems & Solutions

## The Reality

We're proposing to add tools and indexes to an ecosystem that's already inconsistent. Before we add more surface area, we need to address the structural issues—or risk amplifying the mess.

---

## What We Found

### Navigation
- No sticky sub-nav to tie sections together
- Hard to find content (Spending Trends is buried—if we can't find it, users can't either)
- Blog, Resources, and Spending Trends operate as disconnected silos
- "Latest what?" — unclear entry points

### Tagging & Taxonomy
- Inconsistent or missing tags across sections
- Rigid one-to-one URL structure (articles can only live in one category)
- Blog has limited topic filtering; some sections have none
- Filter states on Resources aren't indexable

### Page Consistency
- Some pages have authors, some don't
- CTAs are missing or inconsistent
- No related content or dynamic cross-linking
- Jump links and table of contents appear randomly

### Breadcrumbing
- Unclear where you are within the site
- Can't easily navigate back to parent sections
- Going from case study to blog breaks the trail

### Abandoned Sections
- Podcasts: last episode was 7+ months ago
- Webinars: outdated, not maintained
- Dead ends that hurt credibility

### The Core Issue
Not enough pages. Not enough types of pages. The surface area is limited, and what exists is fragmented.

---

## What We'd Do

### 1. Create a 2-Click Taxonomy
Users can get to any content type within 2 clicks. Everything is findable.

### 2. Sub-Nav Glue
Sticky navigation that ties Blog, Resources, Spending Trends, Tools, and Indexes together. Regardless of where you are, you know what else exists.

### 3. Consistent Breadcrumbing
Always know where you are. Always know how to get back.

### 4. Layer In New Surface Area
Add tools, indexes, and pages the right way:
- Consistent modules (CTAs, authors, dates, related content)
- Dynamic cross-linking with flexible tags
- Hub pages that pull together related content by topic, role, persona

### 5. Prune or Refresh Dead Weight
Audit abandoned sections. Either revive them or remove them.

---

## Problem → Solution Map

| Problem | Solution |
|---------|----------|
| Disconnected sections | Sticky sub-nav that connects everything |
| Unclear breadcrumbs | Consistent breadcrumb system across all pages |
| Rigid tagging | Flexible multi-tag taxonomy with hub pages |
| Not enough pages | Tools + indexes + templates = more surface area |
| Inconsistent page quality | Standardized modules (authors, CTAs, related) |
| No cross-linking | Dynamic related content based on tags |
| Abandoned content | Audit, prune, or refresh |
