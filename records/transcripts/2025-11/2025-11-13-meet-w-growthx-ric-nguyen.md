## Key Topics

### WordPress Publishing Workflow

  - **Drafting:** Authors draft content in external tools (e.g., Google Docs) and paste it into WordPress.
  - **Status Management:**
      - **Draft:** Default status for review; do not publish directly.
      - **Schedule:** Set a future publish date/time.
      - **Publish:** Makes the post live immediately.
  - **Required Fields:**
      - **Featured Image:** Upload a JPEG or PNG file.
      - **Categories:** Select "Blog" plus one other relevant category. If none fit, use "Blog" only.
      - **Meta Description:** Write a short, SEO-friendly description.
  - **Optional Fields:**
      - **Related Posts:** Manually select posts to display, overriding the default automatic selection.
      - **SEO Tools:** Ignore the "Focus keyphrase" field; the meta description and slug are sufficient.

### Content Requirements

  - **TLDR:** Include a 1–2 sentence "Too Long; Didn't Read" summary at the top of every post.
      - **Rationale:** Improves user experience by providing a quick overview for skimmers, matching SpecterOps' existing blog format.

### Permissions & Author Profile

  - **GrowthX Permissions:** Ric granted `pod-0@growthx.ai` `Contributor` access, which allows creating and saving drafts.
  - **Author Profile:** Ric will create a generic "SpecterOps" author profile.
      - **Rationale:** Ensures brand consistency by avoiding attribution to individual GrowthX authors.
