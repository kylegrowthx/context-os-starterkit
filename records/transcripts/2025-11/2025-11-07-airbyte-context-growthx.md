## Summary

## Overview

- **New Audience, New Problem:** Airbyte Context serves AI engineers building agents, not data engineers. This audience struggles with the complexity of building and maintaining data connectors, which distracts them from core product development.
- **Operational Data vs. Analytics:** Agents require real-time, on-demand operational data (e.g., a single customer record) to take action, a fundamentally different pattern from the historical, aggregated data used for analytics.
- **Agentic Data Infrastructure:** Airbyte Context is a new category, providing the data layer that powers agent context. It is the "pipes" that feed internal data to an agent, while the agent itself handles execution and models.
- **Open Source is Critical:** Open source is essential for AI infrastructure. It provides the transparency and self-hosted control required for agents handling sensitive internal data, building trust that SaaS cannot.

---

## Key Topics

### The Problem: AI Engineers Become Accidental Data Engineers

  - AI engineers building agents are forced to become data experts, distracting them from their core product.
  - **Why this is a problem:**
      - **High Maintenance:** Connectors are "living organisms" that break with vendor API changes.
      - **Scaling Issues:** Rate limits and API restrictions cause connectors to fail as user counts grow.
      - **Configuration Complexity:** Customers' unique system setups are often incompatible with generic connectors.
  - **The outcome:** Teams get bogged down in connector maintenance, preventing focus on agent logic and product innovation.

### The Solution: Airbyte Context as Agentic Data Infrastructure

  - **Mission Evolution:** Airbyte's mission remains "making data available and actionable." The change is the consumer: agents, not just warehouses.
  - **Category Definition:** "Agentic data infrastructure"—the data layer powering agent context.
  - **Core Function:** Provide on-demand data access to agents. Airbyte Context is the "pipes" that feed internal data, while the agent handles execution, models, and GPUs.
  - **Analogy:** An agent is like a new employee with a goal. Airbyte Context gives it access to all necessary resources (HubSpot, Google Docs) to complete the task.

### Why Now: The Agent Paradigm Shift

  - **Market Readiness:** The agent market is moving beyond support agents (a proven cost-center) into marketing, engineering, and other core business functions.
  - **Efficiency Gains:** Agents automate tedious tasks, freeing humans for higher-level work (e.g., list enrichment time reduced from weeks to hours).
  - **Technology Readiness:** LLMs are now capable enough to act as the "CPU" for these operational agents.

### The Foundation: Open Source & Enterprise Trust

  - **Why Open Source is Critical for AI:**
      - **Rapid Innovation:** Fosters community discovery of new patterns in a fast-moving field.
      - **Security & Control:** Provides the transparency and self-hosted deployment options required for agents handling sensitive internal data (e.g., HR, finance).
  - **Enterprise Advantage:** Airbyte's background serving banks and insurance companies provides a proven track record for deploying secure, compliant data infrastructure at scale.

### Success Metrics & Go-to-Market

  - **Initial Goal:** 10,000 GitHub stars for the new "AI-ready connectors."
  - **Key Metrics:**
      - **Open Source Adoption:** Usage of AI-ready connectors.
      - **Platform Conversion:** OS users converting to the Airbyte Context platform.
      - **Revenue Growth:** Sales of platform features.
      - **Product Adoption:** Self-serve signups.
  - **Target Customer:** AI-native startups or high-velocity internal teams building agent-powered products.
