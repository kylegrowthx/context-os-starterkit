## Key Topics

### Content Strategy: Goals & Analysis

  - **Partnership Goals:**
      - **Scale Production:** Increase output from 4 posts/quarter to 5/week, mixing new content with refreshes.
      - **Recover Organic Traffic:** Address recent declines by optimizing existing posts.
      - **Build AI Authority:** Establish Redis as a leader in vector databases, semantic caching, and agent memory.
  - **Content Analysis:**
      - **Redis:** Strong in caching, data structures, and vector databases.
      - **Competitors:** Focus on AI infrastructure, vector search, RAG, and agentic AI.
      - **Technical Depth:** Redis content has a good balance. Competitors like Milvus are highly technical, while Pinecone is product-focused with enterprise topics (RAG, security).

### Proposed Content Strategy: 6 Clusters

  - **1. Vector Database Fundamentals:** Build on existing content (e.g., "What is a Vector Database?") with refreshes and new topics.
  - **2. RAG & Retrieval Architecture:** Position Redis as the infrastructure layer for RAG systems, covering the full data pipeline (vector search, keyword search, speed).
  - **3. AI Agents & Agentic Systems:** Position Redis as the tool for fast, persistent memory. **On hold pending Redis messaging playbook.**
  - **4. LLM Optimization & Cost Reduction:** Reframed to **LLM App Optimization**. Focus on performance and accuracy, not just cost.
  - **5. Competitive Comparisons:** Continue creating comparison blogs, listicles, and "alternatives" posts, which drive fast LLM citations.
      - **Process:** GrowthX uses a "product feature matrix" (Notion doc) to ensure accuracy.
  - **6. Top Verticals:** Create content for tech and financial services, linking general topics (e.g., AI for fraud detection) to specific Redis solutions.

### Reporting & Data Alignment

  - **LLM Tracking:**
      - **Problem:** GrowthX uses Scrunch, while Redis uses Profound, creating data silos.
      - **Solution:** Redis will grant GrowthX access to its Profound instance to align tracking.
  - **Performance Metrics:**
      - **Request:** Aida requested MQL data for blog/glossary performance analysis.
      - **Solution:** Redis will provide 12 months of "hand raiser" (meeting bookings) data by subdirectory, which is a strong indicator of intent.
  - **New Dashboard:** GrowthX will build a Looker dashboard merging GSC/GA4 data, including an LLM referral page.

### Content Production Logistics

  - **Featured Images:** GrowthX can provide AI-generated featured images (abstract graphics or text overlays) to streamline publishing.
      - **Process:** GrowthX designer Katya creates a prompt based on Redis brand guidelines → GPT generates 5–10 image options per post.
      - **Decision:** Cody will share this proposal with the Redis design team.
