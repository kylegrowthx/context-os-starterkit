## Overview

- **Automate Post-Processing:** Convert manual Claude edits into automated pipeline steps to improve efficiency and consistency.
- **Use Claude for Customization:** Use Claude to add new steps (e.g., FAQ) by feeding it your pipeline YAML and a working example from another client.
- **Prioritize Artifacts for Tuning:** Store instructions in separate artifacts (700–1k tokens) for easier, non-technical tuning, as editing them is simpler than modifying the pipeline YAML directly.
- **Debug with Flow:** Use the Flow monitor to trace agent decisions and identify bottlenecks, like the fact-checker removing links, which led to a process fix.

---

## Key Topics

### Pipeline Customization & Automation

  - **Goal:** Automate manual post-processing steps (currently done in Claude) by converting them into pipeline steps.
  - **Method:** Use Claude to generate new steps.
      - **Input:** Your pipeline's YAML code + a working example from another client (e.g., Biologica's Editorial V2 pipeline).
      - **Prompt:** Ask Claude to integrate the example step into your pipeline.
  - **Instruction Storage:**
      - **Artifacts (Recommended):** Store instructions in separate files referenced by the pipeline.
          - **Why:** Easier to edit and tune without touching the pipeline's code.
      - **Inline:** Embed instructions directly in the pipeline YAML.
          - **Why:** Requires editing the YAML itself, which is less efficient for frequent adjustments.
  - **Artifact Token Limit:**
      - **Guideline:** Aim for 700–1,000 tokens per artifact.
      - **Why:** Large artifacts can degrade agent performance and cause instructions to be missed.

### Debugging & Optimization with Flow

  - **Tool:** The Flow monitor provides a detailed log of every agent decision.
  - **Use Cases:**
      - **Identify Bottlenecks:** Trace agent reasoning to find where a step is failing or taking too long.
      - **Fix Process Flaws:** The Monograph pipeline's fact-checker was removing links.
          - **Problem:** The fact-checker ran *after* the linker.
          - **Solution:** Move the fact-checker step to run *before* the linker.

### Pipeline Ramp-Up Strategy

  - **Initial Focus:** Operate existing pipelines and perform manual post-processing in Claude.
  - **Next Goal:** Automate repetitive manual corrections by converting them into pipeline steps.
  - **Learning Path:**
    1.  **Observe:** Review complex pipelines (e.g., Biologica Editorial V2) to see advanced capabilities.
    2.  **Iterate:** Add one new step at a time to your existing pipeline to learn the process.
  - **Timeline:** A pipeline typically reaches a stable state after 2–3 batches (2–3 weeks), with initial batches requiring more manual work.
