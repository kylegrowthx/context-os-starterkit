## Key Topics

### Problem: Inefficient QA Workflow

  - The previous QA process was inefficient, requiring editors to manually implement suggested changes from comments.
  - This created unnecessary work and slowed the content pipeline.

### Solution: Direct Technical Edits

  - The new process requires direct edits in Google Docs using "Track Changes."
  - **Procedure:**
    1.  **Make the edit directly** in the document.
    2.  **Add a comment** explaining the rationale for the change.
  - **Scope:**
      - **Focus:** Technical accuracy for a developer audience (e.g., code, workflows, specs).
      - **Out of Scope:** Editorial style, internal links, or non-technical content (e.g., app registration steps).
