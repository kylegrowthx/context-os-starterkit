  - GA4 reports inflated metrics, with sessions and Google search clicks showing significant variances.
  - This is likely a symptom of the root cause: UTM parameter duplication.

### Problem: UTM Duplication

  - UTM parameters are duplicating, causing inflated data.
  - **Example:** The `medium` value is appearing up to three times for a single session.
  - This issue is unfamiliar to both Andrew (10+ years experience) and Talal (16 years experience).

### Context: Site & Tracking Setup

  - **Site:** Next.js headless application.
  - **Tracking:** GTM loads GA4 script in the `<head>` via `app.tsx`.
  - **Consent:** OneTrust manages user consent, which triggers the GTM tags.
  - **Initial Hypothesis:** A Next.js page-loading bug was considered but dismissed as unlikely for a mature framework.
