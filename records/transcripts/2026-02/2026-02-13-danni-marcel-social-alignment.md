## Key Topics

### What's Blocking Marcel from Posting

Marcel has ideas but no system. Three problems:
1. **No calendar time** — 15-minute gaps between meetings don't work; posting needs dedicated 1-2 hour blocks
2. **No visual asset bank** — Marcel is a visual person; without graphics he hesitates. He doesn't know which Figma files to use, what fonts, what visual language
3. **No shared understanding of the strategy** — docs get sent without walkthrough; Marcel needs to see the big picture before acting on details

### Three-Phase Workflow

Marcel proposed separating posting into phases that can have their own calendar blocks:
1. **Brainstorm** — ideas from meeting recordings, repurposing existing content, monitoring sources like Search Engine Journal, Twitter
2. **Shape** — co-develop with Danni; pull examples, build hooks, create assets
3. **Polish and publish** — final pass and post

### Visual Asset Bank

Marcel's biggest immediate blocker. He wants a Figma board he can open and immediately pull from — consistent fonts, colors, graphic styles. Right now he opens Figma and doesn't know if he should use brand guidelines, team project files, or something else. The time spent navigating eats the posting window. He wants his posts treated as "first-class citizens" with the same production support as YouTube/community content.

### Commenting Strategy & Positioning

Danni is building Airtable lists of high-value comment opportunities in three buckets: AEO/SEO space, founder space, and closed deals (to stay in orbit with prospects). Marcel confirmed the direction is right and wants to expand:
- Track and mirror Ethan's commenting — everything he comments on should alert Marcel to also engage
- Insert into AEO/SEO conversations (Kevin Indig, Eli, others) to gain visibility without becoming "the AEO guy" — positioning as a generalist who engages on relevant topics
- Automate alert generation using Airtable + Claude Code + Airtable MCP to surface opportunities in real time

### Post Sourcing

Ideas come from multiple sources: meeting recordings, Twitter trends, Search Engine Journal, existing content repurposing (e.g., Jason's hooks/offers templates), and monitoring what resonates (e.g., "editorial content is most cited" from Profound Post research). The key is feeding ideas into the Brainstorm phase and then shaping them with Danni before committing design time.

### Tools & Automation

- **Airtable:** Central system for content calendar, posting ideas, and commenting opportunities across AEO/SEO/founder spaces
- **Figma:** Visual library organized for quick access (fonts, colors, templates, successful post examples)
- **Notion:** Strategy documentation and high-level planning
- **Claude Code + Airtable MCP:** Automation for surfacing commenting opportunities, monitoring posts, and generating alerts based on Ethan's activity and other key commenters
