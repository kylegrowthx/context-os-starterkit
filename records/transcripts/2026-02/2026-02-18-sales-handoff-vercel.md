### Content Priorities: Gateway + Index

Marcel narrowed the initial scope to two things:

1. **AI Gateway enrichment** — Vercel's gateway index page lists all supported AI models but each model page has only one sentence. Goal: enrich every model page with latency, token usage, input/output specs, cost, and answers to common user questions. Analogy: the MCP catalog GrowthX built for other clients, or the vulnerabilities database. Target: rank #1 for model-specific search queries (e.g., "what is the latency of [model]").

2. **Index / editorial section** — Modeled after Anthropic's content index. Cody created a `/i` section as a backdoor around the CEO's editorial control over the main blog. Still needs to be high quality — Marcel was emphatic that "no slop" is the bar. Study the CEO's posts and every published piece from Vercel to calibrate voice and standards.

### Quality and Speed Expectations

Marcel was direct: this cannot be mediocre. He called out a recent example of a Sprint team client where the blog content was poorly written and convoluted — "I don't know what I'm reading within five seconds — that's not good." With Vercel, quality failure is not recoverable given the CEO's standards and the visibility this account brings. At the same time, speed is everything — Cody's explicit ask. Plan: set up the workspace, ingest all public Vercel docs, study the CEO's Twitter, build the best possible artifacts before the kickoff so the first deliverable looks like GrowthX moved "crazy fast."
