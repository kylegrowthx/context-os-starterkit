### Google Search Console Technique (Search Analytics for Sheets)

- Use Extensions > Search Analytics for Sheets to pull data directly from GSC
- Pulls URL and all ranking queries for each page (up to 25,000 rows)
- Key advantage: groups keywords by page and query (standard GSC export doesn't do this)
- Can set daily or monthly refresh, creating new sheets with fresh data automatically
- More robust than SEMrush because it uses SentinelOne's actual internal search console data

### Google Analytics for Content Titles

- Use Google Analytics custom reports to pull page title and page path for all pages
- SentinelOne has 4,400+ pages, many with titles that hint at keyword intent
- Page titles and URLs are essential for cross-referencing with search console keyword data
- Exported data can be used to feed Apps Scripts for automated comparison

### Apps Script Automation Approach

- Build Apps Scripts using ChatGPT prompts that compare words across URLs, titles, and ranking keywords
- Even without exact matches, scripts can identify related content (e.g., variations, typos, keyword subsets)
- Must use constants (doc IDs, sheet names) so scripts are reusable across accounts
- Always cross-check ChatGPT output; it sometimes over-filters (e.g., removing all instances of "malware detection" when you only wanted one variation)
- Sydney had built a similar script but lost it when she deleted her old workspace; ChatGPT can rebuild it from scratch
