## Overview

- GrowthX uses deterministic workflows over freestyle agents for predictability, with Temporal for orchestration
- Focus on runtime performance and error handling over developer-friendly APIs
- Evaluation and quality control are critical challenges, currently relying on human review
- Recent LLM improvements allow for more agentic approaches and less focus on RAG, but require robust evaluation

---

## Key Topics

### GrowthX's Technical Approach

  - Uses Temporal for workflow orchestration and state management
  - Moved away from RAG to passing full context (up to 5000 words) to newer, more capable models
  - Focuses on deterministic workflows over freestyle agents for predictability
  - Uses TypeScript codebase with native SDKs for various AI services (OpenAI, Anthropic, etc.)
  - Implements parallel processing for efficiency (e.g., splitting article sections)

### Workflow Management and UI

  - Custom-built interface for managing and monitoring workflows
  - Engineers use a more technical interface (based on Temporal) for debugging
  - Content team uses a simplified pipeline view with human-in-the-loop checkpoints
  - Workflow code is generated by an AI agent and then refined by engineers

### Evaluation and Quality Control

  - Currently relies heavily on human review
  - Developing an LLM-as-judge system for automated evaluation
  - Challenges in implementing ML-style evaluations due to engineering team's background
  - Aiming to integrate evaluations into CI/CD pipeline

### Cost and Latency Considerations

  - Less concerned about per-call costs due to B2B model
  - Can tolerate higher latency (minutes vs. seconds) compared to consumer-facing chatbots

### Lessons Learned and Future Directions

  - Newer models allow for more agentic approaches from the start
  - Less focus needed on RAG with improved model capabilities
  - Critical to implement robust evaluation systems early
  - Consider using ML specialists for evaluation system design
