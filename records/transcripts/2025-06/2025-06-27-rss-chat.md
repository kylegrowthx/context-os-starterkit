## Key Topics

### RSS vs. Manual Authoring

  - RSS feeds work well for news-heavy orgs with frequent publishing
  - Manual authoring currently offers more design control in beehiiv's system
  - RSS limitations: less control over spacing, heading sizes; standard H2 \~24px
  - Future development may allow more RSS-specific styling options

### Design Considerations for Email

  - Email uses "1999 technology," limiting some design capabilities
  - Two-column layouts preferred; three columns can be problematic
  - Icons/images require workarounds (e.g., column structure, emojis)
  - GIFs can create more interactive, engaging content than static embeds

### Design Inspiration and Examples

  - beehiiv team has collected strong newsletter examples
  - Highlighted newsletters: Claymation, Coinsnacks, The Mighty, Prof G, The Neuron
  - Web versions of newsletters often available for reference

### Content Display Strategies

  - For varied content (tweets, quotes, videos), consider:
    1.  Two-column layout for balance
    2.  Using GIFs or short video clips to increase engagement
    3.  Screenshots or custom graphics for tweets instead of embeds
    4.  Hyperlinking images/GIFs to source content

### beehiiv's Additional Services

  - CSM packages available for scaling clients
  - Options include dedicated tech support (e.g., Victor for custom solutions)
  - Strategy consulting available with top newsletter experts (e.g., Tyler, DK)
