### Developer Learning Hub Proposal

- Concept: Create a dedicated developer content hub at developers.webflow.com/learn (name still being workshopped)
- Problem statement: Webflow's brand is anchored on visual builder and designers; developers don't immediately understand Webflow as an API/headless CMS platform
- Inspiration: Stripe's developer-focused property, which clearly signals "this is for engineers" through positioning and content
- Content to include: headless CMS tutorials, API documentation, code snippets, technical website building, and content that targets developers in discovery phase (searching for "best headless CMS" before knowing Webflow)
- Rationale: Integration articles capture demand from people already using Webflow ("Webflow + Stripe"); developer hub drives organic awareness from developers not yet familiar with Webflow
- Challenges:
  1. Webflow currently has no distinct home for developer-focused marketing content
  2. Blog and glossary target high-level audiences, not technical builders
  3. Docs exist but are technical reference, not marketing/educational
- Next steps:
  1. Jason to create higher-fidelity mockup and sample content
  2. Anushri to discuss concept with Kirat Chhina (her manager) for approval
  3. Anushri to involve design team to review proposal
  4. Explore technical implementation: Build with Firm, Webflow CMS (as proof point of headless capabilities), or existing stack

### Content Ownership and Resources

- Anushri Gupta oversees Webflow's developer relations team, all developer-facing content on developers.webflow.com, and enterprise developer partnerships (e.g., HubSpot)
- She reports to Kirat Chhina, who leads the entire developer ecosystem (APIs, pricing, packaging)
- Current resources: One documentation lead working with engineering to write technical guides and API reference
- Developer marketing is under-resourced; Webflow team currently creates their own marketing content (blog posts, video series on featured apps)
- GrowthX engagement is explicitly aimed at filling this developer marketing gap
