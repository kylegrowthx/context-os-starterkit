## Overview

- Primary focus: Optimize developer.webflow.com content for SEO and user experience
- Key goals: Position Webflow as a headless CMS, improve discoverability of integrations/APIs
- Target audience: Professional web developers, with secondary focus on hobbyists/prototypers
- Competitors: WordPress, Contentful, Adobe Experience Manager are top priorities to differentiate against

---

## Key Topics

### Current Content Challenges

  - Manual content creation process is not scalable
  - Existing content not optimized for SEO
  - Missing content for key use cases (e.g. Vite/Vue extensions)
  - Some pages not indexed due to client-side JavaScript
  - Need to improve tone, formatting, and structure for SEO

### Expanding Webflow's Perception

  - Not widely recognized as a headless CMS despite capabilities
  - Opportunity to highlight power/flexibility of CMS and APIs
  - Need content showcasing complex use cases and enterprise implementations
  - Educate on migration paths from other CMSs to Webflow

### Developer Audience Insights

  - Professional developers evaluating for large implementations
  - Hobbyists/prototypers using for rapid development
  - Agencies have major influence on CMS selection for enterprises
  - Developer word-of-mouth and conference speakers shape perceptions

### Content Strategy Ideas

  - Create programmatic content for long-tail integration/API topics
  - Develop migration guides addressing common pain points
  - Showcase community examples/galleries of MCP server usage
  - Optimize for developer influencers and conference speakers
  - Improve API documentation to win developer trust

### Measurement and Baselines

  - Establish baseline metrics for developer.webflow.com content
  - Analyze search console data, Zendesk tickets for content gaps
  - Track improvements in search rankings for key terms
