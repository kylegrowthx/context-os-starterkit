## Summary

GrowthX is rolling out Linear as the company-wide project management system to replace scattered Slack threads and Notion pages. Daniel Lopes demoed Linear to the Ops team, showcasing how it handles projects, tasks, milestones, and Slack integration—with emphasis on creating tickets immediately when assigning work. The team agreed to adopt Linear aggressively over the next few weeks, starting with moving the new hires tracker from Notion to Linear and each team member piloting one project or task this week.

---

## Context

The GrowthX Ops team manages company-wide operations—hiring, financial reviews, IT/cybersecurity, MDM tool selection, employee onboarding, and background checks. Work currently scattered across Slack threads, Notion trackers, and Google Sheets makes visibility difficult and creates risk that requests fall through cracks. Bridget McGillivray leads the Ops function and finds herself managing many async requests across multiple channels. Daniel Lopes, who runs engineering/delivery projects, has been using Linear successfully for several months and is championing adoption across the company. The team decided to standardize on Linear as their single source of truth for task management—syncing with Slack to keep communication in place while centralizing work tracking.

---

## Relevance

**To GrowthX Operations:**
- Linear replaces fragmented Slack threads and Notion pages, improving visibility on hiring, financial reviews, onboarding, and background checks. Each task now lives in one place with assignees, due dates, and status tracking.
- Private teams allow sensitive work (hiring offers, finance reviews) to stay confidential while still linking to external docs (Google Sheets, contracts). No sensitive content stored directly in Linear.
- Templates (new hire checklist, month-end reviews, background check steps) can be duplicated for recurring processes. Bridget committed to forcing adoption by adding 2-3 items daily.
- Slack integration creates tickets automatically when assigning work—no manual handoff needed.

**To GrowthX Culture:**
- Shifting from "Slack as source of truth" to "Slack for conversation, Linear for work." Reduces cognitive load on managers tracking async requests.
- Team committed to aggressive adoption: each person pilots one project this week, creates 1 ticket minimum weekly, with Daniel and Yousef demonstrating features (recurring tasks, sub-issues, Slack shortcuts).

**To Operational Risk:**
- Consolidates scattered hiring/finance/onboarding data, reducing risk of offers, approvals, and background checks slipping through cracks. Easier to audit who's responsible for what and when.

## Key Topics

### Linear Overview and Benefits

  - Syncs with Slack for streamlined communication
  - Provides a centralized location for task tracking and project management
  - Offers customizable views, boards, and filtering options
  - Allows for easy prioritization and assignment of tasks

### Ops Team Linear Setup

  - Created a private Ops team in Linear
  - Discussed potential for public and private projects within the team
  - Considered sub-teams for specific areas (e.g., IT and Cybersecurity)
  - Agreed to start with a shared miscellaneous board for small tasks

### Use Cases and Examples

  - Employee onboarding: Create a project template with recurring tasks
  - Background check process: Break down into steps, assign team members
  - Month-end financial reviews: Set up recurring tasks with assignments
  - MDM tool selection and implementation: Create a project with milestones

### Adoption Strategy

  - Each team member to move one project/task into Linear this week
  - Create Linear tickets immediately after Slack communications for task assignments
  - Over-index on using Linear for a few weeks to learn what works best
  - Calibrate priorities together, with Bridget providing oversight

### Privacy Considerations

  - Discussed need for private projects/teams for sensitive information (e.g., hiring, finance)
  - Agreed to link to external documents for confidential content rather than storing in Linear
