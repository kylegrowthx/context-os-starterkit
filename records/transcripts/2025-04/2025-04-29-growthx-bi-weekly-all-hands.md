### Product Demo: Image Generation

  - Problem: Clients request graphics beyond cover images (section breakers, infographics, quote sections with text overlays); currently manual and time-intensive
  - Solution: Two-part workflow using Recraft (image generation) + Claude vision (text placement analysis) + custom style prompts
    - Standalone app: Input any image + text, generates 5+ variations with auto-positioned text at multiple canvas sizes
    - Full article workflow: Pass article URL, Claude identifies illustration opportunities, generates multiple graphic options per opportunity (10 variations), uses Recraft to render, teams pick best versions
  - Style customization: Tested with Zooplo (purple/neon cyberpunk brand aesthetic) — generates variations matching brand identity
  - Current status: Hitting rate limits on vision model due to parallel requests; workflow spreading out requests and tweaking (ready by Thursday/end of week)
  - Future: Exploring chart/graph generation from structured data (HTML generation instead of SVG for chart library compatibility)

### Product Demo: Fact Checker

  - Context: Previous fact-checker ran in Aero Ops with significant scaling limitations (limited parallel checks); Kirkland built core logic, Daniel built app wrapper
  - Infrastructure upgrade: Moved to GrowthX internal infrastructure — now unlimited parallel checks instead of Aero Ops bottleneck
  - Workflow: Parse article → identify fact-checkable items (statements, code snippets, data) → execute against Perplexity search → confidence scores (Likert -2 to +2) → rewrite incorrect claims only
  - Output: Detailed analysis with reason, Perplexity evidence, confidence score, suggested corrections; leaves "unsure" items for human review (e.g., marketing claims, early-stage product announcements where web data is limited)
  - Live demo: Tested on fake article (everything wrong) and Abnormal article (100% accurate); showed chunking, fact extraction, validation, and rewrite
  - Human-in-loop: Editors can manually verify "unsure" items; can also feed customer documents for fact-checking (e.g., customer confirms feature specs)
  - Domain-specific exploration: Testing Kappa.ai (documentation website → Q&A chat) for technical clients like Strappy (Next.js documentation fact-checking); exploring code execution for validating code snippets
  - Open question: Still figuring out code snippet execution and validation for technical clients
