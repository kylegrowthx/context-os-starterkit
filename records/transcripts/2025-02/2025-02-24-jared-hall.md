## Overview

- Attio offers flexible reporting with various chart types and the ability to report on any attribute
- Workflows can be used to enforce data quality and automate processes like lead routing and deal management
- Attio's data model differs from Salesforce, using "people" instead of leads/contacts and treating all potential business as "deals" from the start
- Native integrations (e.g., Typeform) and workflows enable extensive automation of CRM processes

---

## Key Topics

### Reporting Capabilities

  - Attio provides various chart types: line, bar, pie charts
  - Reports can be created on any attribute in the system
  - Examples: ARR ranges, deals in stage, average time in status, sales funnels, forecasted revenue
  - Reports can be customized (e.g., changing from monthly to weekly view)
  - Option to create both shared and private reports

### Data Model and Object Structure

  - Uses "people" object instead of separate leads and contacts
  - Deals exist from the beginning of the funnel, no lead conversion process
  - Deal stages can be used to represent the entire sales process (e.g., lead stage to closed-won)
  - People are automatically synced from email interactions

### Data Quality and Required Fields

  - Currently no native "required fields" feature (in development for early 2023)
  - Workaround: Use workflows to bounce deals back if certain fields are not filled
  - Workflows can provide immediate feedback via pop-up notifications

### Integrations and Automation

  - Native integration with High Touch for data warehouse connections
  - Extensive Zapier integration available
  - API access for custom integrations
  - Typeform integration for capturing website leads
  - Email sequence capabilities for follow-ups

### Lists and Views

  - Lists are subsets of objects, useful for creating specific views (e.g., VC portfolio companies)
  - Can be used when object limits are reached or for one-time events
  - Workflows can automatically add people to lists based on criteria
