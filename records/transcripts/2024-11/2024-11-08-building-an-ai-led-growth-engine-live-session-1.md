## Overview

- The workshop aims to inspire participants to invest time in becoming "AI native" by learning to build and apply AI workflows
- Key concepts covered: identifying use cases, decomposing tasks, prompt engineering, and leveraging different AI models
- Hands-on practice focused on building a basic prospect research workflow using web scraping and AI-powered analysis
- Participants given access to AirOps sandbox environment with pre-built templates to explore further

---

## Key Topics

### Introduction and Workshop Overview

  - Fourth cohort of this workshop, with over 130 previous participants
  - Today's focus: Setting foundation, showcasing possibilities, hands-on workflow building
  - Next week: Deeper dive into advanced concepts and techniques

### Identifying AI Use Cases

  - Look for time-consuming tasks, repetitive work, high-impact areas
  - Consider tasks you'd do if you had more time/resources
  - Analyze calendar to identify opportunities for automation
  - Think about leveraging unique data/knowledge your company has

### Fundamentals of AI Workflows

  - Key components: Inputs, processing steps, variables, planning, critics/adapters, post-processing
  - Importance of decomposing tasks and taking a first principles approach
  - Balance between automation and human review/input

### AirOps Platform Overview

  - Core components: Workflows, Grids, Knowledge Bases
  - Types of workflow steps: LLMs, code, conditionals, human review, data operations
  - Importance of variables for reusability and maintainability

### Hands-on Workflow Building

  - Built a basic prospect research workflow:
    1.  Web scraping of prospect's website
    2.  AI-powered research using Perplexity
    3.  Composing a prospect brief using GPT-4
  - Demonstrated iterative improvement of prompts and outputs
  - Showcased more advanced workflows (e.g. sales email generation, content strategy)

### AI Model Considerations

  - Perplexity: Real-time info, but API limitations
  - GPT-4: Versatile workhorse
  - Claude: Strong performance, good for specific use cases
  - Anthropic AI One Preview: Longer outputs, but slower and more expensive

### Additional Examples and Q\&A

  - Daniel demonstrated a sentiment analysis workflow for client calls
  - Discussed integration possibilities (e.g. Zapier, custom API calls)
  - Addressed questions on data privacy, account setup, and next steps
