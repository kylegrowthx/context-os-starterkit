## Overview

- Webflow admin access via "?edit" URL param; simple UI for content updates
- CMS structure: static pages, collections (blogs, case studies, etc.), direct text edits
- Publishing process: make changes, click "Publish" (blue button) to go live
- Blog categories flexible; avoid "HockeyStack users" for product updates

---

## Key Topics

### Webflow Admin Access

  - Access admin panel: add "?edit" to URL end
  - Login bar appears at bottom of screen
  - Sections: Pages (static), Collections (CMS), Forms

### Content Management

  - Pages tab: 455 total site pages (home, pricing, etc.)
  - Collections: backend CMS (blogs, case studies, reviews, categories)
  - Blog editing: modify title, body, category, URL directly
  - Static page edits: click pencil icon on text/images for inline changes

### Publishing Process

  - Any content change activates blue "Publish" button
  - Click "Publish" to push changes live on domain
  - Forms data bypasses Webflow, connects directly to HubSpot

### Blog Categories

  - Flexible category creation (e.g., content marketing, industry overviews)
  - Avoid "HockeyStack users" category (reserved for product updates)
  - New categories can be added via "+" icon
