## Overview

- Project scope narrowed to focus on assignment management and client collaboration, deprioritizing automated crawling/AI enhancements
- Critical to build client-facing views for content approval and feedback, using passwordless authentication
- Data model centers around Pages, Opportunities, and Assignments, with Pages as the canonical object
- Aim to ship core assignment management functionality in August, integrating with analytics project later

---

## Key Topics

### Current Content Production Challenges

  - GrowthX facing capacity constraints with 6-8 new clients ready to start
  - Inefficiencies from context switching between multiple tools (Airtable, Google Docs, Slack)
  - Manual processes for opportunity identification, assignment creation, and client feedback incorporation

### Revised Project Priorities

  - Focus on assignment management and streamlining workflow
  - Build client-facing views for content approval and feedback
  - Implement basic opportunity and page inventory management
  - Deprioritize automated crawling, AI enhancements, and advanced analytics integration

### Data Model and Core Functionality

  - Pages as the canonical object, including draft/planned pages
  - Opportunities with states: pending, converted, dismissed
  - Assignments with full CRUD, state management, and assignment to team members
  - Client view for reviewing and approving assignments/content

### Client Collaboration Features

  - Passwordless authentication for client access
  - Simplified interface mimicking current Airtable app experience
  - Ability to comment on content, but not directly edit
  - Potential for AI-assisted comment incorporation workflow

### Integration with Other Projects

  - Align with analytics project for page data and metrics
  - Consider future enhancements like persona management and advanced AI workflows
