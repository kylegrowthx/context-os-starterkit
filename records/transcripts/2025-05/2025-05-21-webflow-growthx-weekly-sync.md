## Key Topics

### Content Strategy and Information Architecture

  - Proposed structure: Documentation, Learn (blog), Resources, and Support as top-level categories
  - Separation of API reference from guides within documentation
  - Need for clear distinction between guides (Webflow-specific) and organic content (general developer topics)
  - Glossary suggested as an index page for Webflow terms and structures

### Content Creation and Publishing Process

  - GrowthX to start writing content, focusing on non-cannibalizing topics
  - Content to be temporarily published on main Webflow blog until developer site structure is finalized
  - Vivian to help determine which topics are suitable for blog vs. dev blog
  - GrowthX to set up Webflow workspace for content staging

### Integration Pages

  - Ongoing discussion about the structure and placement of integration pages
  - Potential for creating a standalone subfolder or marketing value page for integrations
  - Need to improve discoverability and value proposition of integrations

### Site Restructuring Challenges

  - Current developer site hosted on Fern, built off a managed Git repository
  - Need to align on a forward strategy to avoid constant reiteration
  - Colin working on finalizing information architecture in the next two weeks

### SEO Considerations

  - Restructuring documentation unlikely to significantly impact SEO due to Webflow's authority on the subject
  - Organic content restructuring could potentially hurt SEO, but currently not an issue due to lack of existing content
