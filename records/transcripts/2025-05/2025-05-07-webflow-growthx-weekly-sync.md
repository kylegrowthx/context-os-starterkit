## Summary

GrowthX and Webflow aligned on two major developer-focused initiatives: (1) integration pages that will live on webflow.com/integrations as a hub for app-based and custom code integrations, designed to serve developers and visual designers building with Webflow, and (2) a new developer publication on developers.webflow.com to host organic, developer-centric content positioning Webflow as a developer-friendly platform. The team agreed on architecture, content structure, and SEO strategy, including separating developer content from the main blog and establishing design approval workflows before launch.

---

## Context

Webflow is a no-code/low-code platform that serves multiple personas: visual designers, developers, and agencies. GrowthX is producing strategic content to expand Webflow's developer mindshare and SEO visibility in technical communities. This weekly sync between Webflow's product and content teams (Kirat Chhina, Vivian Hoang, Anushri Gupta) and GrowthX's content team (Jason Gong, Sydney Go) addresses a core business challenge: most developers view Webflow as a design tool, not as a platform for developer-driven projects. By creating dedicated integration pages and a developer publication, Webflow and GrowthX are building separate experiences for builders versus developers, clarifying content architecture, and improving organic search visibility for developer-intent keywords (e.g., "integrate Stripe with Webflow", "headless CMS"). The initiative also launched Webflow Cloud this week, introducing full-stack application deployment alongside Webflow sites, which expands the developer use case significantly.

## Overview

- Integration pages will live at webflow.com/integrations as a pillar page hub linking to both marketplace app solutions and custom code integration guides, serving developers and visual designers
- A new developer publication will launch on developers.webflow.com to host organic, developer-centric content (e.g., headless CMS guides, API integration tutorials, full-stack development) separate from the main Webflow blog
- Design approval workflow: GrowthX will submit Figma designs matching webflow.com standards; Webflow's design/brand team reviews; approved designs move to development and content population in parallel
- Content differentiation: developers.webflow.com focuses on higher-funnel awareness (e.g., "how to migrate CMS", "developer guide to localization"), while integration pages target lower-funnel, integration-specific intent (e.g., "Webflow Stripe integration")
- SEO strategy includes 301 redirects from outdated University integration articles to new pages, preventing cannibalization by clarifying which content serves which audience (designers vs. developers)
- Webflow Cloud (launched Monday) expands developer positioning—full-stack applications now deployable alongside Webflow sites, creating new integration use cases

## Key Topics

### Integration Pages Structure and Location

  - Will be housed at Webflow.com/integrations
  - Designed as a hub for both app installations and custom code integrations
  - Should cater to visual designers stretching into development and full-stack developers
  - Will link to marketplace apps, custom code examples, and potentially Webflow Cloud for deeper integrations
  - Design inspiration to be drawn from the existing marketplace, maintaining consistency for user experience

### Content Strategy for Integration Pages

  - Focus on comprehensive coverage of integration options
  - Include use cases to improve user engagement and reduce bounce rates
  - Ensure content is SEO-optimized while prioritizing user intent
  - Differentiate from existing app pages to avoid cannibalization
  - Plan to set up redirects from outdated University articles to new integration pages

### Developer-Focused Publication

  - Proposed new blog/publication targeting developers, separate from the main Webflow blog
  - Likely to be housed on developers.webflow.com for SEO benefits and faster implementation
  - Content to focus on developer-centric topics, avoiding overlap with existing blog content
  - Will help position Webflow as a developer-friendly platform, not just a design tool

### Content Differentiation and SEO Considerations

  - Developer content should be distinctly technical, catering to a developer audience
  - Avoid duplicating efforts between developer content and main blog articles
  - Ensure proper canonicalization to prevent SEO conflicts
  - Consider long-term strategy for potentially moving to Webflow.dev domain in the future
