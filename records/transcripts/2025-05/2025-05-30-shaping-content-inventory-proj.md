## Overview

- The MVP should focus on a simplified content inventory system with basic assignment tracking and brief creation capabilities
- Standardize on Markdown for content editing, using TipTap editor components throughout the interface
- Prioritize creating a user-friendly dashboard with key metrics and easy access to content/assignment details
- Consider integrating AI-assisted features for content optimization and contextual guidance, but not as a core MVP feature

---

## Key Topics

### Content Inventory System Architecture

  - Proposed hierarchy: Account \> Workspace \> Projects \> Assignments/Opportunities
  - Core data models: Pages (URLs), Keywords/Prompts, Topics, Clusters
  - Assignments as central records, with associated Brief and Output documents
  - Standardize on Markdown format for content, using TipTap editor components

### MVP Features and Priorities

  - Basic content inventory with URL importing from sitemaps
  - Assignment tracking with simple status/milestone updates
  - Brief creation and editing capabilities
  - Dashboard view with key metrics (e.g., pages tracked, activity snapshot, pipeline overview)
  - Minimal permissioning system to start, focus on core functionality

### User Interface Considerations

  - Aim for a hybrid approach: static dashboard components + dynamic, customizable views
  - Consider integrating an AI chat/prompt feature for assistance, similar to Cursor or GitHub Copilot
  - Explore contextual AI tips for improving specific content areas or fields

### Content Strategy and Organization

  - Topics and Clusters help organize content strategy, but may not be essential for MVP
  - Keywords remain relevant for volume prediction and opportunity assessment
  - Consider future adaptations for AI/LLM-driven content discovery and optimization

### Bylines and Collaborators

  - Track bylines (contributors) for SEO purposes and content attribution
  - Manage internal collaborators (team members, reviewers) for workflow purposes
  - Consider scraping existing byline data during initial content import
