## Overview

- The new PodHQ Notion template, built from Katya's task tracker, will be the standard for all pods.
- **Goal:** Streamline leadership meetings by using pre-work (Weekly Health Assessments) to focus discussions on high-level blockers, not granular tactical updates.
- **Ownership Shift:** Production team members are now responsible for creating their own OOO coverage plans, reducing the coordination burden on EMs and leadership.
- **Key Enhancements:** The template will be updated with a new `Client Meeting Spaces` property, a `Related Linear Ticket` property, and a more intuitive process for adding health assessments.

## Key Topics

### PodHQ Template Review

  - Leah built the PodHQ template in the main hub, incorporating all views from Katya's task tracker.
  - **Key additions:**
      - A `Pod` property in the master task list to enable team-specific filtering.
      - An automated Monday Update button using Katya's existing template.
      - Dedicated views for pod management (e.g., book of business, upcoming renewals).

### Task Management & Workflow

  - Katya's team is not yet onboarded; the tracker currently serves as a personal brain-dump and reference tool.
  - **Proposed Team Workflow:**
      - EMs assign tasks.
      - Production team members check off completed tasks.
      - EMs use the tracker to follow up on outstanding items, replacing manual Slack checks.
  - **Challenge:** Managing tasks that are also Linear tickets.
      - **Solution:** Add a `Related Linear Ticket` property to the task database.
      - **Rationale:** This enables filtering for tasks tied to Linear, while the link within the task page provides full context (e.g., stage, assignee).

### Weekly Health Assessments & Leadership Reporting

  - The `Weekly Health Assessments` column in the Client DB was identified as awkward to use.
  - **Solution:** Hide the column and use a streamlined workflow:
    1.  Click the client page.
    2.  Click the "New page" template button.
    3.  Fill out the assessment directly on the new page.
  - The `Action Items this Week` and `Blockers` fields are for leadership-level summaries, not a tactical to-do list.
  - **Goal:** Use these summaries as pre-work for leadership meetings, enabling focused discussions on high-level issues.

### Out-of-Office (OOO) Coverage

  - The template includes a new process for OOO coverage.
  - **New Ownership:** Production team members are now responsible for creating their own coverage plans.
      - **Rationale:** This reduces the coordination burden on EMs and leadership.
  - The process is designed for a "least common denominator" user (e.g., a new hire) to ensure clarity and self-sufficiency.
