## Overview

- **Production owns final quality.** The production team is the final quality gate; EMs focus on client strategy and expectations. Substandard work erodes client trust and can damage the company's reputation across its interconnected client network because fractional CMOs and other decision-makers often work across multiple clients.
- **Use Linear for systemic fixes.** File tickets for recurring issues (e.g., >30% rewrites) to fix the pipeline, not just patch content. Proactive follow-up on Linear is required to prevent blocks.
- **Human judgment is non-negotiable.** Atlas relies on Claude, a non-deterministic system whose output quality varies daily. The pipeline can produce "filet one day and dog food the next," so human review is essential for consistent quality.
- **New OOO process is self-serve.** Production teammates now create their own coverage plans in Notion for absences >=1 week, finding peer coverage and notifying their EM.
- **SEO MetaTags workflow improved.** The workflow now accepts custom writing guidelines via an artifact and generates three options, addressing past feedback on quality and lack of choice.
- **Time Doctor desktop app is required.** Use the app, not the browser extension, for tracking. The extension records data opaquely and creates timing confusion.

## Key Topics

### Quality & Trust: Production's Responsibility

- **The Problem:** Substandard work is eroding client trust.
  - One poor deliverable can negate goodwill from ten good ones.
  - Poor performance for one client damages the company's reputation with other clients who share the same stakeholders (e.g., fractional CMOs).

- **The Solution:** Production owns final quality.
  - EM Role: Client strategy, expectation management, and high-level feedback.
  - Production Role: Ensure all deliverables meet the client's "standard of good" before submission.
  - Action: Proactively work with EMs to define the standard and create a clear editorial checklist.

- **Human Judgment is Non-Negotiable:**
  - Atlas relies on Claude, a non-deterministic system whose output quality varies daily.
  - The pipeline can produce "filet one day and dog food the next."
  - Human review is essential to maintain a consistent quality baseline and drive continuous improvement.

### Systemic Fixes: When to Use Linear

- **Purpose:** Linear tickets provide data to identify and fix systemic issues in Atlas.
- **Process:**
  1. File Ticket: Document any bugs or recurring problems.
  2. Follow Up: Check your Linear inbox regularly for questions from the engineering team.
  3. Prevent Blocks: Respond promptly to avoid delaying the fix. Tag your EM if client input (e.g., an API key) is required.

- **When to File:**
  - Recurring Issues: If you're consistently rewriting >30% (ideally 40-50%) of an article, the pipeline is inefficient and needs fixing.
  - Strategic One-Offs: If a one-off issue signals a new content type, ask your EM if it will recur. If yes, file a ticket to build a sustainable pipeline.

### New Process: Out-of-Office (OOO) Coverage

- **Goal:** Shift OOO coverage planning from leadership to the production team.
- **Process for OOO >=1 Week:**
  1. Create Plan: Use the "Create a Plan" button in Notion (PodHQ or your Pod page).
  2. Document Details: Include dates, EM, pod, and client.
  3. Add Resources: Link to the client's Operating Manual, Editorial Checklist, Content OS, and Atlas pipeline.
  4. Add Context: Include notes on tone, style, and examples of successful content.
  5. Find Coverage:
     - Post the plan in your pod's Slack channel.
     - Use the "Overview of who might have capacity" table to find a peer.
     - If needed, post a summary in the main #production-team channel.
  6. Notify EM: Inform your EM once coverage is confirmed.

- **Process for OOO <1 Week:**
  - Complete all scheduled work ahead of time.
  - Consider creating a plan for complex situations (e.g., a multi-day absence during a critical delivery window).

### System Updates: SEO MetaTags & Time Doctor

- **SEO MetaTags Workflow:**
  - Improvement: The workflow now accepts custom writing guidelines via a separate artifact (e.g., meta tag guidelines).
  - Output: Generates three options for each meta tag.

- **Time Doctor:**
  - Mechanism: The Atlas pipeline acts as a "writer," pasting task titles into Time Doctor.
  - Limitation: Time Doctor's API prevents renaming individual tasks.
  - Workaround for Title Changes:
    1. Submit a generic list (e.g., "Article 1") to overwrite existing tasks.
    2. Submit the final, correct list.
  - Required App: Use the desktop app for tracking. Do not use the Google Docs browser extension, as it creates opaque data buckets and confuses timing.
  - Support: Mariana can fix errors, such as a pipeline requiring all slots to be filled.
