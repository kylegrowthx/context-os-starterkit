## Key Topics

### Editorial quality + SEO risks (current state)

  - Quality bar below target (≈6/10) → Jon can produce faster than current edit cycle
  - On-page issues → excessive repeated citations; missed content elements vs Ramp; insufficient SEO optimization
  - Keyword selection miss → targeting terms where Brex already ranks top 5/top 10 → keyword cannibalization risk
  - Actionable signals available → GSC/Ahrefs/SEMrush should flag existing rankings; ensure pre-brief checks before drafting
  - Competitive gap → Ramp articles more robust; would outrank current drafts without fixes

### Tools program (highest priority)

  - Goal → create high-volume, high-intent “soft conversion” inventory (e.g., invoice generator/template, QR/PDF utilities later)
  - Launch requirement → minimum 5 tools at go-live so hub doesn’t look thin
  - URL strategy → /tools as parent; all tools nested (e.g., /tools/invoice-template); avoid past Ramp structure
  - Monetization/visibility → supports SEO, interlinks into core funnels, and unlocks paid (ads to tools)
  - Input governance → some tool inputs require internal approvals; build admin fields so Jon can set values post-launch

### Templates hub (second priority)

  - Purpose → capture “{thing} template” demand; convert via downloads/uses; interlink from related articles and product pages
  - IA/UX models → Smartsheet (category pages with multiple variations), Canva (strong template taxonomy + filters), Coefficient (reference; org is “weird”)
  - Structure → consider /templates as parent with topical subfolders (e.g., /templates/expense-management/expense-report, mileage, per-diem)
  - Content pattern → short “how-to” sections on template pages; cross-linking among tools, templates, and guides; “+ template” in meta where appropriate
  - Asset formats → Google Sheets/Excel first (easy to create); evaluate FreshBooks-style pages; ensure CTAs placed in relevant editorial (e.g., expense reporting)

### Pry.co strategy (third priority)

  - Leverage Pry.co’s higher DR/PR links for outsized lift vs microsites
  - Duplicate/selectively adapt tools/templates on Pry.co where strategic; coordinate canonicalization/internal linking to avoid duplication issues
  - “Strip down” Pry.co bloat first; then layer tools/templates; future content play possible

### CMS and platform constraints

  - Sanity latency → publish taking \~45 min (was \~2 min) → investigate and remediate (build hooks, cache/CDN, webhooks, or schema ops)
  - Access mapping → Greg owns web/Sanity; new hire on Core Web Vitals; align permissions, data sources, deployment path to keep engineering out of design release freeze

### Internationalization (phase 2)

  - Emerging need → Dutch pages; separate UK version; explore Spanish (programmatic approach proven on other clients)
  - Considerations → language accuracy, locale routing, hreflang, template/tool localization (e.g., currencies, VAT rates), Spend Trends expansion to ES

### Proof points and guardrails

  - Lovable precedent → 3M impressions, 150 purchases from templates (LTV ≈ $250); strong non-brand utility signal; robust organization and UX
  - Backlinks discourse → avoid bulk/low-quality agency patterns; focus on useful assets that earn links organically

### Resourcing, scope, timeline

  - GrowthX to assign FT engineer; sprint once access unblocked
  - Contract shift → only \~3 weeks left; update language for early conversion or project-based milestone; proceed without delivery risk
  - Delivery cadence → talk to Greg by Thu; provide V1 tools/templates strategy by Thu/Fri/Mon; execute fast post-approval
