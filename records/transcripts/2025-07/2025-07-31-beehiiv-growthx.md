## Key Topics

### Subscriber Onboarding Strategy

  - Create "not initial" segment for remaining 40k subscribers
  - Export CSV, tag in 10k batches (A, B, C, D)
  - Re-import to beehiiv with tags for controlled rollout
  - Aim to add batch before next send (Tue/Wed)

### Web Version and Growth Tools

  - Beehiiv automatically generates SEO-optimized web versions of all posts—no action required, they'll rank for searches like "AI coding," "vibe coding," etc.
  - Use a template like Putty (matches brand colors, displays thumbnails) as the archive landing page; don't worry about cannibalizing Augment Code's site (web archive links back to the main site for SEO backlinks)
  - Create a custom subscribe form in beehiiv Audience > Subscribe Forms, get the embed code, drop it into Augment Code's site footer
  - This funnel sits in the background driving organic lead generation and cross-site backlink value

### Boost Feature for Subscriber Acquisition

  - Access via Grow > Boosts > Get Started > New Offer
  - Requires Stripe Express account connected to beehiiv (Jacob will send setup links)
  - Set your own CPA target (benchmark: $2-$2.50, upper echelon ~$250 max); set spending caps/offer limits if you have fixed budgets
  - Beehiiv's backend verification: geolocation checks, email open history, bounce/spam complaint rates (filters out low-quality addresses)
  - Other publications will request your offer; you approve/reject based on audience fit (use the filter in Monetize > Boosts to see what competitors in your category are paying)
  - Can rotate offers and see which publications drive subscribers, adjust CPA based on performance

### Recommendations Feature

  - Recently launched feature: publications set their top 4 recommended publications on beehiiv (all other beehiiv newsletters)
  - When subscribers sign up, they see recommendations—potential for viral growth across publications
  - Reciprocity mechanic: if you recommend Publication X, they can see you're recommending them, and may recommend you back
  - Strategy for GrowthX multi-client approach: recommend other GrowthX client newsletters + complementary publications; rotate to make room for new clients (e.g., publish new client, add to recommendations, rotate out old client after X weeks)
  - Early data shows strong organic list growth from recommendations feature

### UTM Parameters and Analytics

  - Keep source as "beehiiv" (universal across all posts; don't change per-post—UTM changes must be made in global settings, not per-post)
  - Keep campaign as "beehiiv" (not monthly, because tracking month of send is less valuable than filtering by date range in Google Analytics)
  - For weekly/monthly client reporting: use beehiiv's built-in analytics dashboard (Audience > Reports) to show high-level post performance; use Google Analytics with date filters to see where clicks landed and what happened on Augment Code's site
  - UTM parameters feed into Google Analytics; beehiiv's native analytics are faster for quick client updates
