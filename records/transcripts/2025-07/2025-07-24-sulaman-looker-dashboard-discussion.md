## Key Topics

### Looker Dashboard Setup Process

  - Change data sources from existing client (e.g. DataGrid) to new client (e.g. Flossom)
  - Create custom fields for URL cohorts in both Google Search Console (GSC) and Google Analytics 4 (GA4)
  - Update filters, especially for non-branded queries
  - Ensure cross-filtering is enabled for interlinked chart updates
  - Handle blended data sources carefully, particularly for keyword ranking charts

### Current Manual Processes

  - Weekly updates of newly published URLs required
  - Process involves copying URLs and categories from Airtable to Google Sheets
  - Using Claude AI to format data into case statements for custom fields
  - Updating custom fields in Looker with new URL data

### Automation Opportunities

  - Explore connecting Airtable directly to Looker or using Google Apps Script as intermediary
  - Investigate bulk data source switching capabilities in Looker
  - Consider automating the generation of branded keyword lists for filters

### Additional Dashboard Requirements

  - Most requirements from Andy's ticket are achievable within current setup
  - GSC errors not available via API, may require manual screenshots
  - Content calendar integration possible through Airtable to Google Sheets export

### Hierarchy and Roles

  - Content Managers (CMs) -\> Managing Editors (MEs) -\> Directors -\> Leadership Committee
  - MEs are responsible for presenting reports to clients and updating Looker dashboards
