## Question 1: Company Overview, History, Founding

- Evan You born January 1, 1989; created Vue.js in 2013 while working at Google (consulting role, not full-time)

- Created Vue.js 1.0 in February 2014; called it "something that felt super approachable and very intuitive"
- Raised $1M seed funding from personal angel investors in 2016 while bootstrapping
- Full-time commitment to Vue from 2016 onwards

## Question 5: Product & Feature Analysis

### Core Vue.js Features

**Vue 3 Fundamentals (as of Feb 2026)**
- Composition API: Function-based component authoring (vs. Options API)
- Script Setup: New SFC syntax sugar for shorter, cleaner code
- TypeScript-first internals: Full TypeScript support without extra config
- Reactive system: Proxy-based (vs. Vue 2's object property tracking)
- Single File Components (SFC): Vue, template, and styles in one file
- Fragment support: Multiple root elements per component
- Teleport: Render components outside the DOM hierarchy
- Suspense: Handle async component loading
- Custom Elements: Web components support

**Build Tool Integration**
- Vite (created by Evan You): Instant HMR, 10-100x faster than Webpack
- Vue Loader: Processes .vue files into JavaScript
- Vite Vue plugin: Official Vue 3 support
- SFC TypeScript support: Native `<script lang="ts">` in .vue files

**Ecosystem Tooling**
| Tool | Status | Purpose |
|------|--------|---------|
| **Vite** | Maintained by Evan You | Build tool, dev server, SSR support |
| **Vitest** | Maintained by Evan You | Unit testing framework |
| **VitePress** | Maintained by Evan You | Static site generation |
| **Pinia** | Official state library | Replaces Vuex; simpler, TypeScript-native |
| **Vue Router** | Official | Client-side routing |
| **Vue DevTools** | Official | Browser extension for debugging |
| **Nuxt.js** | Community (now Vercel) | Full-stack meta-framework |
| **Astro** | Community | Static site generator with Vue support |

**Testing & Tooling**
- Vitest: Jest-compatible test runner with HMR
- Vue Test Utils: Official testing library
- Playwright: E2E testing support
- Cypress: Component testing support

### Framework Comparison: Vue vs Next.js

| Dimension | Vue 3 | Next.js | Assessment |
|-----------|-------|--------|-----------|
| **Language** | JavaScript/TypeScript | JavaScript/TypeScript (TS-first) | Parity |
| **Component API** | Composition API (modern) | JSX (React-style) | Vue more intuitive for templates |
| **Server-Side Rendering** | Via Nuxt, Vite SSR, or custom | Native (App Router) | Next.js: built-in, optimized |
| **Build Tool** | Vite (Evan You's) | Next.js proprietary | Vite: faster, more standard |
| **Deployment** | Agnostic; runs anywhere | Vercel-optimized (but portable) | Vercel: optimized for Next.js |
| **Edge Functions** | Via framework (Nuxt, Remix) | Native Vercel Edge Functions | Vercel: advantage |
| **Meta-framework** | Nuxt (community, now Vercel) | Next.js (framework-platform integration) | Vercel owns Next.js |
| **DX (Developer Experience)** | Excellent; lower learning curve | Good; higher learning curve (JSX) | Vue: simpler to learn |
| **Type Safety** | Full TypeScript support | JSX + TypeScript | Parity |
| **Performance** | Excellent (especially with Vite) | Excellent (Turbopack, Server Components) | Parity for most use cases |
| **Community** | 10M+ developers (Asia-dominant) | 20M+ developers (global) | React/Next.js larger |
| **Enterprise Adoption** | Strong in Asia; growing in US | Dominant in US/Europe | Regional differences |
| **Scaling** | Excellent with Nuxt | Excellent with Next.js + Vercel | Vercel platform lock-in advantage |
