- Current status: Subsidiary of Thales Group (December 4, 2023)

## 6. Product Portfolio & Core Features

### WAAP (Web Application & API Protection) Platform

**Core Components:**

1. **Cloud WAF (Web Application Firewall)**
   - Cloud-based protection for web applications and APIs
   - Blocks OWASP Top 10 attacks in real-time
   - Near-zero false positives
   - Managed rules written and tested by Imperva Threat Research team
   - 94%+ customers deploy in blocking mode from day one
   - 97% default blocking rate
   - Integration with Google Cloud, AWS, Azure

2. **Advanced Bot Protection**
   - Multi-layered bot detection
   - Protects against web scraping, account takeover, transaction fraud, DDoS, competitive data mining
   - Defends against OWASP 21 Automated Threats
   - Two deployment models: single-stack (with WAF) or connectors (AWS, Cloudflare, F5, NGINX, Fastly)
   - Cutting-edge techniques with human expertise

3. **DDoS Protection**
   - Layers 3, 4, 7 protection
   - 3-second SLA for DDoS mitigation
   - Global network with 95% sub-50ms latency globally
   - User Behavior Analytics + Threat Intelligence
   - Volumetric and application-layer attack mitigation
   - Network of Super PoPs in Internet connectivity hotspots

4. **API Security**
   - Continuous API discovery (public, private, shadow APIs)
   - OWASP API Security Top 10 vulnerability detection
   - Business logic attack blocking (BOLA, etc.)
   - Integration with Kong, MuleSoft, Azure APIM, Apigee, F5
   - Real-time risk assessments

5. **Runtime Application Self-Protection (RASP)**
   - From Prevoty acquisition
   - Detects and responds to attacks within applications
   - Bridges network edge to application layer to databases
   - Out-of-path architecture for always-on availability

6. **Attack Analytics**
   - Machine learning-driven correlation of thousands of alerts
   - Incident narratives with attack origin, methods, severity
   - Advanced traffic profiling at network edge
   - Incident response and forensics

### Deployment Options

- **Cloud (SaaS):** Cloud WAF managed via Imperva Management Console
- **Hybrid:** Cloud + on-premises appliances (SecureSphere Gateway Appliances)
- **On-Premises:** WAF Gateway Appliances (7th generation hardware)
- **Connectors:** Integration with Cloudflare, AWS, F5, NGINX, Fastly

### Enterprise Features

- **Management Console:** Centralized management across all deployments
- **Infrastructure as Code:** Terraform provider for automation
- **Enterprise Services:** Continuous management by Imperva security experts
- **Managed Services:** On-demand security expertise and monitoring
- **SLA:** 3-second DDoS mitigation SLA
- **Compliance Ready:** Supports SOC 2, ISO 27001, PCI DSS, HIPAA, GDPR

## 12. Competitive Positioning vs. Vercel

### Direct Competitors (Same Security Layer)

**Cloudflare:**
- 300+ PoPs vs. Imperva's smaller network
- More affordable pricing ($20-$10,000 annual range)
- Integrated ecosystem (WAF, DDoS, bot, API)
- Shorter DDoS mitigation (implied faster at scale)
- Easier onboarding and implementation

**AWS WAF:**
- Limited to AWS ecosystem
- Basic API security (rate limiting only)
- No native API discovery
- Pay-as-you-go pricing, cheaper for small-scale
- Less sophisticated threat detection

**Akamai:**
- Massive global network
- Deep behavioral analysis capabilities
- Enterprise-focused (premium pricing)
- Long history with Fortune 500s
- Complex deployment models

### Adjacent Competition (Deployment Platforms)

**Vercel (Frontend Cloud):**
- WAF as secondary feature (not primary product)
- Optimized for deployment, not security operations
- Lighter-weight security model
- Zero-config DX vs. Imperva's enterprise config
- Different buyer persona (developers vs. security teams)

### Key Differentiators

**Imperva Strengths vs. Vercel:**
1. **Dedicated WAF expertise:** 20+ years WAF-specific R&D vs. Vercel's infrastructure-first approach
2. **Enterprise security:** SOC operations, managed rules by threat research teams
3. **Hybrid deployment:** On-prem + cloud flexibility vs. Vercel's cloud-only
4. **DDoS mitigation:** 3-second SLA vs. Vercel's layer 7 DDoS (less proven)
5. **RASP capability:** Runtime protection beyond network edge (Prevoty integration)

**Vercel Strengths vs. Imperva:**
1. **Developer experience:** Git push to deploy vs. WAF configuration complexity
2. **AI integration:** v0, AI SDK not matched by Imperva
3. **Framework optimization:** Next.js integration unique to Vercel
4. **Performance:** Edge-first architecture with minimal cold starts
5. **Simplicity:** Zero-config deployment vs. Imperva's enterprise setup
