## 5. Product & Feature Analysis (60+ sources)

### Core Offerings

**1. Fastly CDN**
- 70+ global Points of Presence
- Varnish-based VCL configuration language
- Origin Shield (second-layer cache to protect origin)
- Real-time purging and analytics
- Image optimization (real-time transforms)
- Streaming media delivery optimized

**2. Compute@Edge (Edge Serverless)**
- Compiled to WebAssembly (WASM)
- Lucet runtime: 35.4 microsecond startup (100x faster than alternatives)
- Supports: Rust, JavaScript, Go, C, Python
- 300-second execution timeout
- No cold starts (warm instances)
- Real-time code execution at edge POPs
- Fully isolated execution per request
- $50/month free credit for Compute

**3. Fanout (Real-Time Messaging)**
- Pub/Sub at the edge
- WebSocket replacement for real-time APIs
- Use cases: Chat, gaming, collaborative editing, live data streams

**4. DDoS Protection**
- Layer 3/4/7 protection
- 497+ Tbps mitigation capacity (as of Sept 2025)
- Real-time Attack Insights (new, April 2025)
- Event-level granularity
- Automatic mitigation rules

**5. Image Optimization**
- On-demand format conversion (AVIF, WebP, JPEG, PNG, GIF, JPEGXL, SVG, MP4)
- Real-time transforms
- Automatic caching of optimized versions
- Per-format analytics

**6. Observability & Analytics**
- Real-time metrics dashboards
- CDN-level analytics
- Request/bandwidth tracking
- Performance dashboards
- Custom metrics

### Technology Stack

**VCL (Varnish Configuration Language)**
- Domain-specific language
- Designed for conciseness and speed
- Used for caching logic, routing, request/response manipulation
- Executes on both edge POPs and Origin Shield

**WebAssembly & Lucet**
- Custom-built Lucet compiler/runtime in Rust
- Ahead-of-time (AOT) compilation via Cranelift code generator
- Extremely fast instantiation (microseconds vs milliseconds)
- Full isolation between requests
- Memory-efficient (few kilobytes overhead)

## 6. Pricing & Packaging (30+ sources)

### Pricing Model

**Structure:** Primarily usage-based (pay-as-you-go), with package options available

### CDN Pricing

**Bandwidth:** ~$0.12/GB for first 10TB (North America/Europe), ~$0.28/GB in Brazil
**Requests:** ~$0.0075 per 10,000 requests
**Minimum:** ~$50/month
**Volume Discount:** ~30% discount after first 10TB consumed

**Premium Services:**
- TLS/HTTPS support
- WebSocket support
- Image optimization add-on

### Compute@Edge Pricing

- **Free Tier:** $50/month free credit
- **Per-Request Model:** Usage-based, priced per execution
- **Serverless Model:** Similar to AWS Lambda, charged for compute time

### Package Options

- Flat-rate packages starting at $1,500/month (for predictable costs)
- Custom enterprise pricing available

**Comparison to Vercel:**
- Vercel: Pro at $20/user/month + usage credits
- Fastly: Usage-based starting at $50/month
- Fastly generally more expensive for small/medium traffic, competitive at scale

## 12. Weaknesses & Criticisms (20+ sources)

### Technical Limitations

1. **VCL Learning Curve:** Requires understanding of Varnish concepts; not beginner-friendly
2. **Limited Node.js Support:** Compute@Edge uses WASM; traditional Node.js workloads not ideal
3. **Package Size Limits:** Compute packages have max size restrictions
4. **Library Compatibility:** Only WinterTC-compatible npm modules supported
5. **Backend Isolation:** Shared IP space means native secrets ineffective; requires header-based auth
6. **App Router Support:** Next.js App Router not yet fully supported on Fastly

### Market Limitations

1. **Brand Awareness:** Less recognized than Cloudflare or AWS among developers
2. **Ecosystem Size:** Fewer integrations, plugins, and third-party tools
3. **Documentation:** Less comprehensive than larger competitors
4. **Community Size:** Smaller user base means fewer tutorials, Stack Overflow answers
5. **Pricing Transparency:** Hard to predict costs; scale unpredictably with usage

### Strategic Weaknesses

1. **Positioning Ambiguity:** Neither pure CDN nor pure developer platform; "in between"
2. **Not a Full Platform:** Lacks deployment automation, version control integration (git-native)
3. **No AI/ML Story:** Limited thought leadership in AI code generation or infrastructure
4. **Smaller Market Share:** 5.5% vs Cloudflare's 40% in CDN category

## 13. Competitive Advantages (20+ sources)

### vs Vercel

1. **Real-Time Capabilities:** Fanout pub/sub has no Vercel equivalent
2. **Mature Streaming:** Purpose-built for media delivery; Vercel not focused here
3. **Performance (Edge Compute):** 35.4 microsecond startup vs Vercel's 50ms+ (estimated)
4. **Pricing Flexibility:** Can be cheaper at very high scale
5. **Multitenancy:** Can run multiple customers/use cases; Vercel is more single-tenant by design
6. **DDoS at Scale:** 497+ Tbps mitigation capacity; Vercel relies on Cloudflare backend

### vs Cloudflare

1. **Streaming Optimized:** Purpose-built video delivery; Cloudflare's edge is more general
2. **VCL Power:** Fine-grained caching control; Cloudflare Workers is simpler but less flexible
3. **Real-Time (Fanout):** Unique capability; Cloudflare lacks equivalent
4. **Performance:** Faster TTFB on certain benchmarks
5. **Real-Time Analytics:** Lower latency insights vs Cloudflare

### vs Akamai

1. **Developer Experience:** Modern, programmable; Akamai seen as legacy
2. **Speed:** Faster edge compute startup times
3. **Pricing:** More affordable for mid-market
4. **Modern Architecture:** WebAssembly vs Akamai's XML-based configs

### Absolute Strengths

1. **WebAssembly Foundation:** Future-proof, secure, fast execution model
2. **Real-Time at Scale:** Proven ability to handle millions of concurrent connections (gaming, chat, streaming)
3. **Enterprise Proven:** Used by major media companies, proven in production at scale
4. **Open Source Lucet:** Community trust, transparency, contribution opportunities
