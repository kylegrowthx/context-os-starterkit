**Architecture Evolution:**
- Classic (2018-2025): Multi-tenant Kubernetes clusters on Google Cloud; pods run containers
- Flex (October 2024-present): Self-hosted, home-grown abstraction layer; VM-based (not Kubernetes)
  - Each developer gets their own remote VM
  - Declarative APIs inspired by Kubernetes control theory
  - AWS-based for Flex (primary infrastructure)
  - Can deploy in <3 minutes
- Decision to leave Kubernetes: Stateful dev environments, unpredictable resource usage, complex permissions, extensive support burden

**Security Features:**

- OIDC support: OpenID Connect for gating access to sensitive resources
- Role-based access: Members vs. Collaborators with granular permissions
- Policy enforcement: Organizational policies for security governance
- AI safety: Isolated containers prevent AI agent damage to host machine
