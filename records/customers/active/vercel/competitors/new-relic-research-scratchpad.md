## 1. Company Overview: Founding, History, Key Milestones

### Founding Story
- New Relic was formed in Delaware as "New Relic Software, LLC" in September 2007
- Converted to Delaware corporation and changed name to "New Relic, Inc." in February 2008
- Founded by Lew Cirne, a serial founder who previously exited Wily Technology (acquired by CA Technologies)
- Cirne spent brief time as Entrepreneur in Residence at Benchmark Capital before founding New Relic
- Company name is an anagram of founder's name
- Launched first commercial product in 2008
- Rebranded from BitBalloon to Netlify in 2017-2018 (Note: this is Netlify, not New Relic; separate company)

**Sources:**

### Founding Timing and Context
- Founded during early wave of Application Performance Monitoring (APM) tools
- Initial product focused on automatic code instrumentation for performance monitoring
- Targeted Java and .NET developers initially, expanded to Node.js, Python, Ruby, PHP, and Go

**Sources:**

## 5. Product & Feature Analysis

### Core Platform Components

**1. Application Performance Monitoring (APM)**
- Automatic code instrumentation (no code changes required)
- Distributed tracing across services
- Trace sampling with tail-based sampling
- Error tracking and analysis
- Support for Java, .NET, Node.js, Python, Ruby, PHP, Go, and more
- <3% performance overhead
- Real-time anomaly detection

**Key APM Features (as of 2025):**
- Code-level insights: See exact line of code causing latency
- Intelligent sampling: Capture the data that matters
- Service maps: Visualize service dependencies
- Error profile: Automatic error clustering and diagnosis
- Deployment markers: Correlate deploys with performance changes

**Sources:**
- https://newrelic.com/blog/apm/what-is-apm
- https://docs.newrelic.com/docs/apm/new-relic-apm/getting-started/introduction-apm/
- https://newrelic.com/platform/application-monitoring
- https://www.saasworthy.com/product/new-relic-apm

**2. Browser Monitoring (Real User Monitoring / RUM)**
- Captures real user experience from 100M+ global users daily
- Core Web Vitals (LCP, INP, CLS) monitoring
- AJAX and HTTP request tracking
- User interaction timing
- Geographic and device-level insights
- Sankey diagrams for user flow visualization
- Session replay capabilities
- Open-source browser agent

**Key RUM Metrics:**
- Page load time (actual, not synthetic)
- Time to interactive (TTI)
- Largest Contentful Paint (LCP)
- Cumulative Layout Shift (CLS)
- Interaction to Next Paint (INP)
- Apdex (Application Performance Index) for user satisfaction

**Comparison to Vercel Speed Insights:**
- New Relic: Full-stack observability with browser as one component
- Vercel Speed Insights: Focused specifically on Core Web Vitals, simpler UX
- New Relic provides more backend context; Vercel provides simpler frontend-only dashboard

## 6. Pricing & Packaging

### Pricing Structure

**Tiers:**

1. **Free Tier (Perpetual)**
   - 100 GB data ingest per month
   - 1 full platform user (unlimited querying, dashboarding, alerts)
   - Unlimited basic users (read-only queries, dashboards, alerts)
   - Access to all platform features: APM, infrastructure, browser, mobile, logs, traces, etc.
   - No credit card required
   - No expiration

2. **Standard/Pro Tier**
   - Starts at $49-99/month for individuals
   - Per-user pricing: $99-129/user/month (estimates vary)
   - Additional data: $0.40/GB after free allowance
   - All platform features included

3. **Pro with Data Plus**
   - Additional $0.60/GB for premium data tier
   - Longer retention periods
   - Higher ingest limits

4. **Enterprise**
   - Custom pricing
   - Volume discounts
   - Dedicated support
   - SLA guarantees
   - Advanced security and compliance features

### Pricing Advantages & Disadvantages

**Advantages:**
- Most generous free tier in the industry (100 GB/month is substantial)
- Unified pricing: One price gives access to 50+ capabilities
- No per-feature charges (unlike Datadog)
- Transparent per-GB rates once free tier exhausted

**Disadvantages:**
- Per-user pricing ($99/user/month) creates friction for larger teams
  - 50-person team: $4,950/month in user licenses alone
  - 100-person team: $9,900/month minimum
- Data ingest charges discourage comprehensive monitoring
  - Teams resort to aggressive sampling (1-10%) to control costs
  - Limits the value proposition for high-data environments
- Unpredictable costs for fast-growing companies
  - Data volume can spike unexpectedly
  - Pay-as-you-go pricing increases quarterly

**Comparison to Datadog:**
- Datadog: Modular pricing per product (APM, logs, infrastructure, etc.) - often $100+/host/month
- New Relic: Unified pricing but higher per-user costs
- Splunk: Volume-based pricing, potentially very high at scale
