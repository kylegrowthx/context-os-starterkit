### Edge Computing (Magic Containers)

- **Docker-based deployment**: Any containerized application anywhere

- **Hardware**: 5GHz+ CPUs, pure NVMe storage per node
- **Integration**: Works with Edge Scripting, Magic Containers, Bunny Database

### Edge Scripting (Deno Runtime)

- **Language**: JavaScript/TypeScript native support (no config)
- **Runtime**: Built on Deno (v8 isolates, Rust-based)
- **Deployment**: GitHub auto-deployment with CD pipeline
- **SDK**: @bunny.net/edgescript-sdk on npm
- **Use cases**: APIs, contact forms, request routing at edge
- **Pricing**: Pay-as-you-go, scales to zero

**Sources:**
- https://bunny.net/edge-scripting/
- https://bunny.net/blog/introducing-bunny-edge-scripting-a-better-way-to-build-and-deploy-applications-at-the-edge/

- **Foundation**: Built on libSQL (Turso's SQLite fork)

- **Integration**: Works with Edge Scripting, Magic Containers, jamstack
- **Pricing**: Pay-as-you-go, only charged for storage when idle
