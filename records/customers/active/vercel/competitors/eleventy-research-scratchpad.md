## Question 1: Company Overview — Founding Story, History, Key Milestones, Headquarters, Mission

### Founding & Creation

**Creator:** Zach Leatherman, a web developer in Omaha, Nebraska.

**Year Founded:** 2017 (first release December 2017)

**Inspiration:** Eleventy was inspired by Jekyll (Ruby-based) but created as a JavaScript equivalent, addressing the need for a simpler, more flexible static site generator.

**Name Origin:** Leatherman's grandmother mispronounced the number "eleven" during his childhood, leading to the name "Eleventy" (abbreviated 11ty).

**Key Distinction:** Eleventy does not require JavaScript frameworks — zero client-side JavaScript by default. It's HTML-first, not JavaScript-first.

**Mission & Philosophy:** Build a "simpler static site generator" that remains zero-config to start while offering unlimited customization. Support multiple template languages (Liquid, Nunjucks, Markdown, Handlebars, Pug, EJS, Haml, WebC, JSX, TypeScript, MDX, and more).

### Key Milestones

- **2017:** First version released in December 2017
- **2020:** Adoption by major projects and government agencies
- **2022:** Won Google Open Source Peer Bonus Award (one of 154 winners from 80+ projects)
- **2024:** Eleventy v3.0.0 released in October 2024 with full ESM support
- **May 2024:** First-ever 11ty Conference held
- **2024:** Moved to full independence via Open Collective funding (goal: $6,000/month)
- **2025:** Continues as fully independent project with community funding

### Headquarters & Organization

- No traditional corporate headquarters (open source project)
- Creator: Zach Leatherman (currently listed as "Builder for the Web" at multiple affiliations: Font Awesome, CloudCannon)
- Maintained primarily by Zach Leatherman with 2+ open source maintainers collaborating

## Question 2: Funding & Financials

### Funding Model

Eleventy is **not venture-backed** in the traditional sense. It operates as a community-funded open source project.

**2024 Independence Fundraising:**
- Goal: $6,000 monthly operating costs
- Model: Open Collective crowdfunding
- First major fundraising push in 7 years of project history
- Transition from corporate sponsorship (Netlify, Font Awesome, CloudCannon backing) to grassroots community support

**Historical Backing:**
- Netlify: Sponsored full-time development at various points
- Font Awesome: Employment of creator Zach Leatherman
- CloudCannon: Current employment of creator

### Revenue Model

Eleventy is completely free and open source (MIT licensed). No revenue is generated by Eleventy itself; the project is funded through community donations on Open Collective and indirect support from companies where Leatherman is employed.

### Headcount

- Primary maintainer: Zach Leatherman (1 person)
- Contributing collaborators: 2+ open source maintainers
- No full-time dedicated team (unlike Netlify, Vercel, Gatsby)

### Financial Status

No public financial data available. This is a community-funded open source project without traditional venture funding, corporate valuation, or revenue targets.

### Key Contrast with Vercel

- Vercel: $863M funded, $9.3B valuation, ~$200M ARR, ~874 headcount
- Eleventy: Community-funded, no VC backing, volunteer-driven maintenance, estimated $6K/month operating costs
- Eleventy business model: Sustainability through community contributions, not growth-at-all-costs venture model

## Question 5: Product & Feature Analysis

### Core Platform Capabilities

**Template Language Support (40+ languages):**
- Default: Liquid, Nunjucks, Markdown, JavaScript, HTML
- Optional: WebC, Sass, TypeScript, Vue, Svelte, JSX, MDX, Handlebars, EJS, Pug, Haml, TOML (via addons)
- Unique strength: True language flexibility vs. Next.js (React-only) or Gatsby (GraphQL + React)

**Build & Deployment:**
- Zero-config by default (no `.eleventy.js` required for basic use)
- Flexible directory structure (vs. opinionated frameworks)
- Watch mode for local development
- Incremental builds (faster rebuilds)
- Passthrough copy for static assets

**Performance Features:**
- Zero client-side JavaScript by default (major differentiator)
- Lightweight output (Eleventy 11ty.dev: 325.5KB total, 9 network requests vs. typical SPA bloat)
- Syntax highlighting via PrismJS (zero-JS)
- Image optimization available via plugin
- Build performance: ~2.6 seconds for 1000 Liquid templates (competitive with Hugo)

**Plugin Ecosystem:**
- Official plugins: Image, is-land, Render, i18n, RSS, Syntax Highlighting, Navigation
- Community plugins: 50+ in Plug11ty registry
- Plugin model: Function-based, lightweight, composable

**Content Management:**
- Data files: YAML, JSON, JavaScript (11tydata.json directory files)
- CMS-agnostic: Works with Sanity, Contentful, Netlify CMS, Decap CMS, Payload, etc.
- Webhooks support for automatic rebuilds on CMS updates

**Advanced Features (v3.0):**
- Full ESM support (modern JavaScript standard)
- Image optimization transform
- @11ty/import CLI for importing content from external sources
- @11ty/create CLI for cross-platform file creation
- Streaming builds (for large sites)

### Key Differentiators vs. Vercel/Next.js

| Feature | Eleventy | Vercel/Next.js |
|---------|----------|----------------|
| **Language Flexibility** | 40+ template languages | React/JavaScript only |
| **Default JavaScript** | Zero client-side JS | Heavy JS framework-dependent |
| **Configuration** | Zero-config default | Config required for optimization |
| **Framework Lock-in** | Framework-agnostic | Next.js optimized |
| **Pricing** | Free (open source) | Freemium ($20/user for Pro) |
| **Deployment** | Any static host | Vercel recommended (vendor lock-in) |
| **Use Case** | Static/JAMstack content | Full-stack, AI, dynamic apps |
| **Build Speed** | Fast for static sites | Better for dynamic/SSR |
| **AI Features** | None | v0, AI SDK, AI Gateway |
| **Serverless** | Not available | Native (Fluid Compute) |

### Enterprise Features

**Eleventy lacks:**
- Native serverless functions (can use Netlify/Vercel functions via deployment)
- Built-in authentication (use third-party: Clerk, Auth0, Supabase)
- Built-in databases (integrate with Neon, Supabase, Fauna, etc.)
- WAF, DDoS protection (delegated to hosting provider)
- Enterprise SLA (available through hosting provider)

**Eleventy includes:**
- Full source control (git-based workflows)
- Audit trail (all builds tracked in git)
- Security: MIT licensed, transparent codebase, community-audited
- Scalability: Handles 1000+ page sites efficiently

### Pricing

- **Cost:** $0 (completely free, MIT licensed)
- **Revenue model:** Community-funded ($6,000/month Open Collective goal)
- **Commercial use:** Unrestricted (MIT license allows any use)
- **vs. Vercel:** Vercel charges $20/user/month Pro, Hobby is non-commercial only

## Question 6: Pricing & Packaging

### Pricing Structure

**Eleventy:**
- **Cost:** Completely free
- **License:** MIT (permissive open source)
- **Commercial use:** Fully permitted (no restrictions)
- **Hosting costs:** Pay separately to hosting provider (Netlify, Vercel, Cloudflare, GitHub Pages, etc.)

**vs. Vercel:**
- Hobby (Free): Non-commercial use only, 1M edge requests, 100GB bandwidth
- Pro: $20/user/month, 10M edge requests, 1TB bandwidth
- Enterprise: Custom pricing (~$20-25K/year minimum)

### Tier Breakdown

Eleventy doesn't have pricing tiers because it's not a SaaS product. The entire feature set is available for free, unlimited:
- Unlimited builds
- Unlimited template languages
- Unlimited plugins
- Unlimited sites
- Unlimited traffic/bandwidth
- Unlimited storage (subject to git repository limits)

### Deployment Costs (Separate)

When deploying Eleventy, users choose hosting independently:

**Free Options:**
- Netlify Free: 300 minutes/month build time
- GitHub Pages: Unlimited
- Cloudflare Pages: Unlimited
- GitLab Pages: Unlimited

**Paid Options:**
- Netlify Pro: $19/month
- Vercel Pro: $20/user/month
- Cloudflare: Pay-as-you-go

### Enterprise Model

Eleventy has no traditional enterprise offering. The project is community-funded at $6,000/month.

For enterprise users wanting managed Eleventy hosting:
- **CloudCannon** offers Eleventy-optimized hosting and CMS
- **Sanity** offers enterprise CMS with Eleventy integration
- Custom deployments via Netlify, Vercel, AWS, etc.

### Competitive Advantage on Pricing

This is Eleventy's strongest positioning against Vercel:
1. **Zero cost** to use (vs. Vercel's $20/user minimum)
2. **No vendor lock-in** on hosting (deploy anywhere static sites work)
3. **Commercial use unrestricted** (Vercel's free tier prohibits commercial use)
4. **Framework freedom** (no lock-in to Next.js/Vercel optimization)

### Value Proposition

- For individuals, agencies, government: Eleventy + Netlify/CloudflarePages is typically cheaper than Vercel
- For teams building Next.js: Vercel is worth the premium for optimization
- For teams building multi-framework: Eleventy is cheaper and more flexible

### Key Differentiators

1. **Open Source & Free:** vs. Vercel's commercial model
2. **Framework Agnostic:** vs. Vercel's Next.js optimization
3. **HTML-First:** vs. JavaScript-first frameworks
4. **Community Funded:** vs. VC-backed growth mentality
5. **Independent:** Recently achieved full independence (2024)
6. **Performance:** Fast for static sites; not competitive for full-stack/AI apps

### Competitive Positioning

**Eleventy wins when:**
- Building static/content-driven sites (blogs, documentation, marketing sites)
- Cost is a primary concern
- Teams use multiple frameworks
- Government/accessibility is a priority
- Zero client-side JavaScript is desired
- Vendor lock-in is a concern

**Vercel wins when:**
- Building Next.js applications
- AI tooling is needed (v0, AI SDK)
- Full-stack, dynamic rendering required
- Enterprise features (SSO, compliance) needed
- Performance optimization for complex apps
- Team wants opinionated, fast setup

### Market Positioning

Eleventy and Vercel are in **different markets**, not direct competition:
- Eleventy: Static Site Generation / JAMstack
- Vercel: Full-Stack / AI-Powered / Next.js

However, they do compete for:
- Websites (blogs, marketing sites, documentation)
- Developer mindshare (Next.js vs. Eleventy for web dev)
- Deployment hosting (Netlify/Cloudflare often preferred for Eleventy over Vercel)
