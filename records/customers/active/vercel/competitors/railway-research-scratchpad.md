## 1. Company Overview & Founding Story

### Founding & History

- Founded in 2020 by Jake Cooper, a Canadian software engineer with experience at Uber, Bloomberg, and Wolfram.
- Jake Cooper studied software engineering at the University of Victoria.
- At Wolfram, Cooper worked on developer experience improvements including linting, Docker, and CI/CD.
- At Uber, he scaled infrastructure for Jump bikes and scooters until Uber sold the division to Lime.
- After leaving Uber, Cooper identified a core infrastructure bottleneck: deploying even simple applications required deep infrastructure knowledge and large DevOps teams.
- Railway was built to solve this bottleneck by making deployment simple, affordable, and accessible to all developers.

### Mission & Vision

- "An all-in-one intelligent cloud provider" for deploying applications, services, and databases.
- Focus on developer experience and reducing infrastructure burden through intelligent automation.
- Platform uses containerized services with zero-config deployment.

### Key Milestones

- 2020: Founded
- 2022 (May): Raised $20M Series A from Redpoint Ventures
- 2023: Killed free tier; pivoted to usage-based pricing model
- 2025 (August): Launched Model Context Protocol server for AI agents
- 2025 (Late): ~2M global users, processing 25M monthly deployments
- 2026 (January 22): Raised $100M Series B led by TQ Ventures

## 5. Product & Feature Analysis

### Core Platform Capabilities

#### Deployment Model

- **Docker-based containerization** - Any application that runs in Docker works on Railway
- **Auto-build from source** - GitHub, GitLab, Bitbucket repositories auto-deploy on push
- **Nixpacks** - Alternative builder (deprecated but supported for legacy services)
- **Railpack** - New default builder for automatic dependency detection and builds
- **Zero-config deployment** - Automatic detection of language/framework/Dockerfile

#### Services & Infrastructure

- **Containerized services** - Deploy any service: web apps, APIs, workers, databases
- **Persistent volumes** - High-performance storage for stateful applications
- **Private networking** - Fast internal service-to-service communication via *.railway.internal
- **Auto-scaling** - Horizontal scaling by increasing service replicas
- **Resource allocation** - CPU (up to 24 vCPU per replica on Pro) and memory (up to 24GB per replica)

#### Managed Databases

- **PostgreSQL** - Zero-config provisioning with automatic connection strings
- **MySQL** - Full MySQL support with one-click setup
- **MongoDB** - Native MongoDB deployments
- **Redis** - Managed Redis for caching/queues
- **Automatic backups** - Built-in backup capabilities for data protection

#### Environments & Preview Deployments

- **Staging environments** - Create parallel, isolated environments for PR testing
- **Preview PR environments** - Automatic ephemeral environments for every pull request
- **Focused PR environments** - Deploy only affected services and dependencies (reduces waste)
- **Automatic cleanup** - PR environments deleted when PR is merged/closed

#### Observability & Monitoring

- **Log Explorer** - Central log search/filtering across all services
- **Metrics dashboards** - Auto-generated graphs for CPU, memory, disk usage, network egress
- **Custom metrics** - Support for application-level metrics via structured logging
- **Alerts & webhooks** - Email alerts and Slack/Discord integration for threshold breaches
- **OpenTelemetry integration** - Send traces to Loki/Tempo/Prometheus via Grafana stack

#### CI/CD & Automation

- **Deploy hooks** - Trigger deployments via webhooks from external systems
- **Environment variables** - Per-deployment configuration management
- **Secrets management** - Secure credential storage
- **GitHub Actions** - Third-party CI/CD integration support
- **Automatic deployments** - Push-based workflows with instant propagation

#### Enterprise & Security Features

- **SAML SSO** - Single sign-on via Okta, Auth0, Microsoft Entra ID, Google Workspace
- **Role-based access control (RBAC)** - Environment-based granular permissions
- **Audit logs** - Complete change tracking for compliance
- **Workspace 2FA enforcement** - Multi-factor authentication requirements
- **Data Processing Agreement (DPA)** - GDPR compliance support
- **SOC 2 Type II** - Security/trust certification
- **SOC 3** - Privacy certification
- **HIPAA compliance** - Health data protection (Enterprise tier)

#### AI-Native Features (2025+)

- **Model Context Protocol server** - Launched August 2025
- **AI agent integration** - Claude and other AI coding agents can deploy/manage infrastructure
- **Direct infrastructure control** - AI agents can call deployments, analyze infrastructure automatically
- **Lightning-fast deployments** - Sub-1-second deployment speed for AI-generated code
- **GPU support** - Planned integrations for machine learning workloads

#### Template Marketplace

- **1,800+ pre-built templates** - One-click deployment for popular applications
- **Open source support** - Revenue sharing with template creators

## 6. Pricing & Packaging

### Pricing Tiers

#### Free Trial
- **One-time grant:** $5
- **Expiration:** 30 days
- **Usage:** Applied toward any platform resources
- **Best for:** Experimenting, personal projects, evaluation

#### Hobby Plan
- **Monthly cost:** $5/month subscription
- **Included credit:** $5/month toward resource usage
- **Billing:** Pay $5 base + any usage above $5 in same month
- **Resource limits:** Minimal; suitable for low-traffic side projects
- **Best for:** Indie hackers, personal projects, learning

#### Pro Plan
- **Monthly cost:** Variable based on usage
- **Entry point:** $5-$20/month for typical projects
- **Resource pricing:** Actual CPU, memory, bandwidth, storage consumption
- **Scaling:** Linear with usage; no surprises from idle resources
- **Best for:** Production applications, teams, growing startups

#### Enterprise Plan
- **Custom pricing** - Negotiated based on requirements
- **SLA guarantees** - Reliability commitments
- **Dedicated support** - Direct access to Railway engineering
- **HIPAA/compliance** - Enhanced security/compliance features
- **Custom contracts** - License and service agreements
- **Best for:** Large organizations, regulated industries, mission-critical applications

### Billing Model

- **Usage-based pricing** - Pay for actual consumption (CPU, memory, egress bandwidth, storage)
- **No idle tax** - Services put to sleep after 10 minutes of inactivity don't incur compute charges
- **Subscription tier as floor** - Your tier's monthly fee is the minimum; pay more if you exceed
- **Predictable scaling** - Linear cost growth with traffic, not exponential spikes
