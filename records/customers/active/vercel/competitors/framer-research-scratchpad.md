## 1. COMPANY OVERVIEW — Founding Story, History, Key Milestones, Mission

### Founding & Early Evolution

- **Founded:** 2014 in Amsterdam, Netherlands
- **Founders:** Koen Bok and Jorn van Dijk
- **Previous venture:** Sofa (design studio, acquired by Facebook in 2011)
- **Pre-Framer background:** Both founders left Facebook in late 2013 to build "something for the next 10-20 years"
- **Initial product:** JavaScript library for code-based prototyping (heavy code dependency)
- **Strategic pivot (2018):** Realized high-fidelity interactive prototyping would not replace static design; pivoted toward **full website creation platform**
- **Product evolution:** Framer Classic → Framer X → Framer Web → Framer (current rebrand)
- **Rebrand timing:** 2018 with Series B positioned company as design-to-website tool, not design-tool-adjacent

### Key Milestones

- **2014:** Founded by Koen Bok and Jorn van Dijk
- **November 2018:** Series B funding ($24M) from Atomico, signaling strategic pivot to website creation
- **2023:** Hit $10M in revenue; raised $27M Series C
- **August 2025:** Raised $100M Series D at $2B valuation (became unicorn)
- **2025 trajectory:** $25M → $50M ARR (100% YoY growth); targeting $100M ARR by end of 2026
- **2025 product launches:** Workshop (AI component generation), enhanced analytics, vector editing tools

### Geographic & Cultural Context

- **Headquarters:** Amsterdam, Netherlands
- **Leadership:** Koen Bok (CEO, founder, technical)
- **Team composition:** 565 employees as of August 2025
- **Customer base:** Originally builders, designers, and startups; expanding into enterprise (40% of Y Combinator Summer 2024 batch uses Framer for main websites)

### Mission & Positioning

**Company narrative:** "The design-to-website platform for the modern web."

Framer positions itself as:
- **Design-first, code-optional:** Visual builders can create without touching code; developers can extend with React/JavaScript
- **AI-native from the ground up:** "AI-era production tools" — design-to-code automation, not design-to-prototype
- **Fast shipping without DevOps:** Built-in hosting, CMS, analytics, SEO — no external deployment/infrastructure
- **Modern, approachable alternative to Webflow + traditional development:** Faster than Webflow, easier than Next.js

## 4. KEY ACQUISITIONS & PARTNERSHIPS — What Framer Bought, Why, Outcomes

### Acquisitions

**Finding:** Framer has made **no notable acquisitions in the 2023-2025 period** that shaped competitive positioning.

- **Historical:** Popmotion acquisition (January 2019, pre-scope of this brief) — animation library for Framer Motion
- **Current strategy:** Organic development of all core features (AI, CMS, hosting, analytics) rather than M&A

### Strategic Partnerships & Integrations

| Partnership | Type | Purpose | Status |
|-------------|------|---------|--------|
| **Figma import** | Native integration | Figma designs → Framer canvas | Active (plugin available) |
| **Figma-to-HTML plugin** | Tool/plugin | Figma → Framer HTML export | Active |
| **Agency Program** | Channel | Free Enterprise access + 50% referral commission | Active |
| **Marketplace integrations** | Third-party ecosystem | Form submissions, SEO, hosting optimizations | Limited (mostly native) |
| **AI provider integrations** | Model connections | OpenAI, Anthropic, Google (via Framer AI) | Emerging (AI features launched 2025) |

### Figma Relationship (Strategic Import)

Framer explicitly enables **Figma-to-Framer workflow**:
- Native Figma plugin allows copying components from Figma → Framer canvas
- Framer auto-translates Figma's Auto Layout → Framer Stacks
- Position: "Figma for design, Framer for production" (not either/or)
- This is Framer's strategic differentiation vs. Webflow (which has no Figma native integration)

### No Vendor Lock-In Story (Unlike Vercel's Next.js)

- **React Export plugin:** Converts Framer designs to React components
- **Self-hosted capable:** Can export and host Framer sites elsewhere
- **Code accessibility:** All generated code is readable, standards-based HTML/CSS/React
- **Implication:** Framer is seen as "easier to exit from" than Vercel (which bundles Next.js framework with deployment)

## 5. PRODUCT & FEATURE ANALYSIS — Complete Feature Inventory, Gap Assessment vs. Vercel

### Core Platform Overview

**Framer positioning:** "Design-to-website platform" with AI, built-in CMS, hosting, and React component support.

**Vercel positioning:** "Frontend cloud" for Git-push deployment, edge compute, and Next.js optimization.

### Feature-by-Feature Comparison Matrix

| Category | Framer | Vercel | Gap Assessment |
|----------|--------|--------|----------------|
| **Design Canvas** | Full visual editor (drag-drop + code) | None (code/CLI first) | **Framer unique** |
| **AI Site Generation** | "Start with AI" (full site generation) | v0 (component-level generation) | Different scope: Framer does whole sites, v0 does UI components |
| **Hosting** | Built-in (auto-deploy on save) | Built-in (git-push deploy) | **Parity** — different workflows |
| **CDN / Global Distribution** | Built-in fast hosting | 126 PoPs + 19 compute regions | **Vercel advantage** — larger edge network |
| **CMS** | Built-in relational CMS | None (use Contentful, Sanity, etc.) | **Framer unique** |
| **Deployment Model** | Visual builder save → live | Git push → automatic deploy | Different paradigm |
| **React Component Support** | Code components, React export plugin | Native framework (Next.js owner) | **Vercel advantage** — framework-platform integration |
| **Edge Functions** | Not available | Yes, V8 isolates at 126 PoPs | **Vercel unique** |
| **Preview Deployments** | Yes (per iteration) | Yes (per PR with unique URLs) | **Parity** |
| **Analytics** | Built-in server-side analytics | Web Analytics (privacy-focused, real-user data) | **Parity** |
| **Security/Compliance** | SOC 2, HIPAA available | SOC 2, ISO 27001, PCI DSS, HIPAA, GDPR, TISAX, DPF | **Vercel advantage** — broader compliance |
| **Performance Optimization** | Auto-optimized (images, code splitting, caching) | Automatic (Fluid Compute, edge caching, ISR) | **Vercel advantage** — more sophisticated optimization (Fluid Compute, RSC) |
| **Build Caching** | Limited | Intelligent build caching | **Vercel advantage** |
| **AI Integration** | Native Framer AI (site generation, component generation, rewriting) | v0 (UI generation), AI SDK (unified API), AI Gateway (multi-provider) | **Vercel advantage** — broader AI stack (SDK + Gateway) |
| **Framework Support** | Framer-optimized for Framer sites; can export to React/Next.js | 40+ frameworks natively optimized | **Vercel advantage** — framework flexibility |
| **Serverless Compute** | Limited (runs on Framer's hosted infra) | Fluid Compute (high-concurrency, 800s execution, minimal cold starts) | **Vercel advantage** — more powerful compute |
| **Database / Storage** | None native | Partners with Neon (Postgres), Upstash (Redis), Vercel Blob (R2) | **Vercel advantage** — more data options |

### AI & Site Generation Deep Dive

**Framer AI (2025):**
- **Start with AI:** Single prompt → full homepage (header, visuals, CTA, animations, color palette)
- **Shuffle feature:** Regenerate layouts/colors without losing content
- **Rewrite tool:** AI rewriting for text sections
- **Workshop (Spring 2025):** Generate custom components/animations via AI assistance
- **On-Page Editing:** Edit live sites directly in browser
- **Vector editing:** AI-assisted vector tools (new in 2025)

**Vercel v0 (2025):**
- **Component generation:** Describe UI → v0 generates React + shadcn/ui + Tailwind CSS
- **GitHub import:** Iterate on real codebases with AI assistance
- **VS Code editor:** Direct code editing alongside AI
- **Branching/PRs:** Full git workflow inside v0
- **Database integration:** Connect to Snowflake, AWS databases
- **One-click deploy:** Deploy to Vercel production

**Competitive positioning:**
- **Framer:** "Full website in minutes" — end-to-end site builder + AI
- **Vercel v0:** "Component in seconds" — UI code generation for developers
- **Market choice:** Designers/PMs → Framer; developers → v0

### React & Code Component Support

**Framer's approach:**
- Code components are custom React components you write in Framer's editor
- Uses Framer Motion for animations (same library Vercel/Next.js developers use)
- Can reuse across projects
- React Export plugin converts designs to React components
- **Not full code-first:** Canvas is visual, code is supplemental

**Vercel's approach:**
- Framework-agnostic git-based deployment
- Supports 40+ frameworks natively
- v0 generates full Next.js component code
- Code-first workflow (code in editor, deploy via git)
- **Not visual:** No canvas/design tool

**Implication:** Framer and Vercel operate in different phases of development. Framer = design-to-prototype-to-website. Vercel = code-to-deploy-to-edge.

## 6. PRICING & PACKAGING — Tier Breakdown, Free Tier, Enterprise Signals

### Current Pricing Structure (2025 Revision)

Framer simplified from 5 tiers to 3 main plans in October 2025:

| Tier | Price | Billing | Audience | Key Inclusions |
|------|-------|---------|----------|----------------|
| **Free** | $0 | — | Exploration, non-commercial | 10 CMS collections, 1,000 pages, 5MB uploads, 1 locale, unlimited form submissions |
| **Basic** | $10/mo | Monthly/Annual | Personal, hobby, side projects | Custom domain, AI tools, fast hosting, SEO, **limited to 1 CMS collection** (reduced from 2) |
| **Pro** | $30/mo | Monthly/Annual | Professionals, small teams, startups | Everything from Basic + staging, instant rollback, roles/permissions, multiple locales, **relational CMS**, site redirects |
| **Scale** | Custom | Annual only | Growing teams, variable needs | Custom limits, dedicated support, annual billing, unlimited capabilities |
| **Enterprise** | Custom | Annual only | Large orgs, strict compliance | Custom hosting, enterprise-grade security, dedicated support, unlimited features, SLAs |

**Pricing evolution note:** October 2025 rebranding reduced Basic from $15/mo → $10/mo (price reduction); Pro from $20/mo → $30/mo (feature bundling). Free tier gained unlimited form submissions and stronger feature parity with Basic.

### Pricing Positioning vs. Vercel

| Metric | Framer | Vercel | Positioning |
|--------|--------|--------|-------------|
| **Free Tier** | Commercial use allowed, generous | Non-commercial only, hard limits | Framer: more accessible entry |
| **Pro tier** | $10-30/mo | $20/user/mo | Framer: flat-rate, per-site; Vercel: per-seat |
| **Enterprise** | Custom | Custom (~$20-25K/year est.) | Both: negotiated |
| **Billing model** | Flat-rate tiers | Usage-based (Pro) + fixed team seats | Framer: simpler; Vercel: scales with headcount |

### Competitive Pricing Intelligence

1. **Framer vs. Webflow:**
   - Framer Basic: $10/mo
   - Webflow Basic: $18/mo
   - Framer wins on affordability for entry-level

2. **Framer vs. Wix:**
   - Framer targets designers/developers
   - Wix targets small businesses/SMBs
   - Non-overlapping pricing tiers (Wix higher for e-commerce, Framer lower for web design)

3. **Usage-based concerns:**
   - Framer does NOT appear to have overage charges (flat tiers)
   - Vercel: Usage-based beyond $20 credit (Pro)
   - Framer message: "Simpler billing, easier to forecast"

### Revenue Per Customer (ARPU) Estimate

- $50M ARR ÷ 45-55K customers = ~$909-1,111 ARPU
- Implies mix of free (non-paying), Basic ($10), Pro ($30), and Scale/Enterprise (custom)
- Estimated customer mix: 50-60% free, 20-30% Basic, 10-15% Pro, 5-10% Scale+

6. **Competitive threat to Vercel:** LOW DIRECT — different value prop, audience, market

1. **Design-first UX:** Vercel targets code-first developers; Framer targets design-first creators

6. **Technical depth:** Vercel's Fluid Compute, RSC, ISR, streaming SSR are more sophisticated than Framer's infrastructure
7. **Ecosystem maturity:** Vercel's Next.js ecosystem is larger; Framer's design community is growing but younger
