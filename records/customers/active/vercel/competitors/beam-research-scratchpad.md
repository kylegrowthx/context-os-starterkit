### 1. Company Overview — Founding, History, Key Milestones

**Founding Story & History:**
- Beam was founded in 2021 by Luke Lombardi (CTO, hardware/software engineer who likes robotics and ML) and Eli Mernit (CEO, former competitive cyclist and hackathon winner)
- They were college roommates who had been building products together
- Before Beam, they frequently attended hackathons and built a serverless framework to quickly ship web apps onto the cloud
- When they started working with AI, they realized no serverless framework existed to make it easy to deploy AI models on the cloud
- Beam is an open-source alternative to Modal for running AI apps

**Company Profile:**
- Y Combinator portfolio company (YC W22)
- Based in New York, NY, USA
- Headcount: 5-7 employees as of 2024; recent hiring for 4 roles
- Revenue: $1M revenue with a 7-person team in 2024 (per GetLatka)

**Key Details:**
- Formerly known as Slai.io, evolved from end-to-end training+inference to inference-focused
- Beta9 is the open-source engine powering Beam Cloud (AGPL-3.0 license)
- 1,555 GitHub stars on main Beta9 repository; 136 forks; actively maintained as of Jan 2026

### 4. Product & Feature Analysis

**Core Platform Capabilities:**
- **Sandboxes:** Securely execute code generated by LLMs in Sandboxes
- **GPU Inference:** Deploy serverless GPU inference endpoints
- **Background Jobs:** Task queues, scheduled jobs, long-running functions (no timeouts)
- **Serverless Compute:** Run on CPUs and GPUs with instant autoscaling
- **Container Deployment:** Launch any Docker container in the cloud in under 1 second
- **Multi-Cloud Support:** Self-hostable via Beta9 (AGPL-3.0 open source)

**Performance Metrics:**
- Cold starts: 2-3 seconds for most functions
- Warm starts: 50-100ms
- Container launch time: Under 1 second (via custom runc runtime)
- Persistent sandboxes: No 45-minute limit (unlike Vercel Sandbox) — can run indefinitely until terminated
- Execution time: No built-in timeout limits for background jobs

**Developer Experience:**
- Python-first and TypeScript SDKs
- Hot-reloading support
- CLI-based workflow
- No YAML configuration required
- Pythonic interface for AI workloads

**GPU Support:**
- GPUs: NVIDIA A100, H100, 4090s and others
- Pay-per-millisecond billing
- No upfront costs or commitments
- Free tier: 15 hours of GPU credit

**Pricing:**
- CPU: $0.190 per core
- RAM: $0.020 per GB
- File storage: Included free
- Pay-per-second (or per-millisecond for billing granularity)
- No published GPU rate card (custom configurations)

**Framework & Runtime Support:**
- Python (primary)
- TypeScript/Node.js (beta)
- Docker (deploy any custom image)
- Language agnostic via Docker

### 8. Competitive Positioning

**Direct Competitors:**
1. **Modal** ($200M+ valuation, backed by Sequoia, more mature, proprietary, 2-4s cold starts)
2. **RunPod** (more cost-flexible, broader hardware options, BYOC approach, 48% cold starts <200ms)
3. **E2B** (code sandbox focus, microVM-based for code execution)
4. **Vercel Sandbox** (newer entrant, 45m execution limit, Firecracker microVMs, integrated with Vercel ecosystem)

**Adjacent Competitors:**
- AWS Lambda (general serverless, no GPU native)
- Google Cloud Run (general serverless)
- CoreWeave (GPU clusters, not serverless-focused)
- Replicate (pre-trained models, higher latency)
- Northflank (full-stack platform with GPU orchestration)

**Beam's Competitive Advantages:**
1. **Open-source + self-hostable** — Beta9 can be deployed anywhere (on-prem, multi-cloud, hybrid), avoiding vendor lock-in
2. **Ultra-fast cold starts** — 2-3 seconds, competitive with Modal, faster than many RunPod workloads
3. **Developer-centric UX** — Python SDK, hot-reloading, no YAML config
4. **Pay-per-millisecond granularity** — Fine-grained billing
5. **Unlimited session duration** — Persistent sandboxes vs Vercel's 45-minute limit
6. **GPU support from day 1** — Not a bolted-on feature like Vercel Sandbox

**Beam's Competitive Weaknesses:**
1. **No vendor differentiation** — Not as tightly integrated with a popular framework like Vercel+Next.js
2. **Limited analyst coverage** — Pre-analyst-coverage stage startup
3. **Small team (7 people)** — Limited resources for GTM, ecosystem development
4. **Early-stage funding** — Only $4M raised vs Modal's $80M+ trajectory
5. **Minimal enterprise positioning** — No SOC 2, HIPAA, or compliance certifications mentioned
6. **Low brand awareness** — Unknown outside serverless GPU niche
7. **Commodity compute** — Can't compete on price alone vs RunPod's raw GPU access
8. **No killer app** — Unlike Vercel's Next.js ecosystem or Modal's MachineLearning community

## Summary: Beam's Role vs Vercel

**Beam is NOT a direct Vercel competitor in the traditional sense.** It's a specialized serverless GPU platform for AI workloads (inference, training, sandboxed code execution). Vercel is a frontend cloud platform optimized for web applications.

**However, there is emerging competitive overlap:**
1. **AI Sandbox Execution:** Both now offer isolated environments to run untrusted/AI-generated code
   - Beam: Container-based, unlimited session duration, GPU-capable, multi-cloud
   - Vercel: Firecracker-based, 45m-5h limit, no GPU, ecosystem-integrated
2. **AI Workload Deployment:** Vercel is expanding into AI with v0, AI SDK, and Sandbox; Beam is expanding upmarket
3. **Developer Experience:** Both prioritize ease of use and fast iteration
4. **Pricing:** Vercel's opacity on AI workloads creates opportunity for Beam's transparent pricing

**Strategic Positioning:**
- Beam is a **niche player in serverless GPU**, competing with Modal and RunPod
- Vercel is a **broad platform** extending into AI, competing with Netlify, Cloudflare, AWS Amplify
- Beam + Vercel could be **complementary** (Vercel for frontend, Beam for GPU inference backend)
- Beam could **threaten Vercel's Sandbox** if Vercel customers need GPU support or multi-cloud flexibility
