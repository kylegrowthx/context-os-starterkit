# Sync Handbook

Syncs all content from the GrowthX Handbook (growthx-handbook repo / handbook.growthx.ai) into `docs/handbook/` in growthx-context.

---

## Paths

| Item | Path |
|------|------|
| Script | `.cursor/skills/sync-handbook/scripts/sync-handbook.py` |
| Output | `docs/handbook/` |
| Manifest | `docs/handbook/.sync-manifest.json` |
| Private docs | `docs/private-docs/` (content with no handbook equivalent) |
| Archive | `docs/archive/pre-handbook-sync/` (replaced originals) |

## Prerequisites

- Python 3.8+
- Both `growthx-handbook` and `growthx-context` repos accessible on disk

## Triggers

- "sync handbook"
- "update handbook docs"
- "pull handbook changes"
- "handbook to context"

## Workflow

### 1. Run the sync script

```bash
python3 .cursor/skills/sync-handbook/scripts/sync-handbook.py \
  --handbook-path "/Users/marcelsantilli_mac2/Dropbox/GrowthX AI/Projects/growthx-handbook" \
  --context-path "/Users/marcelsantilli_mac2/Dropbox/GrowthX AI/Projects/growthx-context"
```

Default behavior: only syncs files whose source hash changed since last run.

**Flags:**
- `--dry-run` — show what would change without writing
- `--force` — full rebuild regardless of hashes
- `--verbose` — detailed logging
- `--migrate` — one-time migration (splits existing docs into private-docs + archive)

### 2. Review output

The script reports new, updated, deleted, and unchanged file counts. Review the changes.

### 3. Update root README and INDEX

After syncing, update these files to reflect any structural changes:
- `docs/README.md` — overview of the three-folder structure
- `docs/INDEX.md` — references to handbook/, private-docs/, archive/
- `docs/handbook/README.md` — auto-generated by the script
- `docs/handbook/INDEX.md` — auto-generated by the script

### 4. Verify

- Spot-check a few synced files to confirm metadata blocks are correct
- Confirm `source` URLs point to `https://handbook.growthx.ai/{path}`
- Confirm JSX components (CardGroup, Tabs, etc.) are preserved

## What the script does

1. Scans all `.mdx` files in growthx-handbook (excludes context/, scratchpad/, .cursor/, node_modules/)
2. Replaces YAML frontmatter with a `<metadata>` block:
   - `purpose`: from the description field
   - `source`: full live URL (https://handbook.growthx.ai/{path})
   - `sync_type: auto`
   - `access: build-team`
   - `last_synced`: today's date
3. Preserves all JSX/Mintlify components as-is
4. Renames `.mdx` to `.md` (keeps original filename, no -v1 suffix)
5. Writes to `docs/handbook/` preserving directory structure
6. Auto-generates README.md and INDEX.md for every directory
7. Writes `.sync-manifest.json` tracking source hashes for incremental sync
8. Copies image assets from the handbook

## Edge Cases

- **First-time setup**: Use `--migrate` flag to split existing docs before syncing
- **Handbook file deleted**: Next sync removes it from docs/handbook/ and updates INDEX
- **Manual edits to handbook/**: Will be overwritten on next sync (the rule warns about this)
- **New handbook directories**: Auto-created with README/INDEX
