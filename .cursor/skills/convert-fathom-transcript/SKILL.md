# Convert Fathom Transcript

Convert raw JSON data from `projects/fathom-backup/` into formatted `.md` transcript files in `records/transcripts/`.

This is a **two-phase** process:
1. **Script phase** — deterministic JSON-to-markdown conversion (no LLM needed)
2. **LLM enrichment phase** — generate Context and Relevance sections, fix metadata

This skill is for **Fathom** data only. For Fireflies meetings, use the `pull-meeting` skill.

## Inputs / Outputs

| | What | Example |
|---|---|---|
| **Input** | Meeting name, month, or "all" from user | "Convert the fathom meetings from June 2025" |
| **Script** | `projects/fathom-backup/convert-to-transcript.py` | -- |
| **Output** | Formatted transcripts in records | `records/transcripts/2025-06/2025-06-30-ed-jason-weekly.md` |

## Source Data

Each meeting in `projects/fathom-backup/YYYY-MM/YYYY-MM-DD-meeting-slug/` contains:

| File | What's inside |
|------|--------------|
| `metadata.json` | Title, dates, URLs, recorded_by, participants |
| `summary.json` | Fathom AI summary as markdown (Meeting Purpose, Key Takeaways, Topics, Next Steps) |
| `transcript.json` | Speaker-labeled transcript array with timestamps |
| `transcript-raw.json` | Same transcript from dedicated endpoint (fallback) |
| `action-items.json` | Action items with assignees and timestamps |
| `participants.json` | Calendar invitees with emails and external flags |

## Phase 1: Script Conversion

### Single Meeting

```bash
cd projects/fathom-backup
python3 convert-to-transcript.py 2025-06/2025-06-30-ed-jason-weekly
```

### Batch by Month

```bash
python3 convert-to-transcript.py --month 2025-06
```

### All Downloaded Meetings

```bash
python3 convert-to-transcript.py --all
```

### Preview Without Writing

```bash
python3 convert-to-transcript.py --all --dry-run
```

### Overwrite Existing Files

```bash
python3 convert-to-transcript.py --month 2025-06 --force
```

### What the Script Produces

The script reads JSON files and writes a `.md` file with this structure:

```markdown
# Meeting Title

<metadata>
date: YYYY-MM-DD
time: HH:MM UTC
duration: X minutes
organizer: email@example.com
participants: Name1, Name2, Name3
fathom_recording_id: 12345678
fathom_url: https://fathom.video/calls/...
share_url: https://fathom.video/share/...
source: fathom
enriched_on: (added by enrich-fathom-transcript skill)
</metadata>

---

## Summary

<from Fathom's "Meeting Purpose">

---

## Context

_To be generated by LLM._

---

## Relevance

_To be generated by LLM._

---

## Overview

- Key takeaway bullets from Fathom

---

## Key Topics

### Topic Name

- Details from Fathom summary

---

## Action Items

**Person Name**
- Action item text

---

## Transcript
**Speaker Name:** Dialogue text.

**Another Speaker:** Response text.
```

**Key differences from Fireflies transcripts:**
- Metadata uses `fathom_recording_id`, `fathom_url`, `share_url` instead of `fireflies_id` and `transcript_url`
- Includes `source: fathom` tag
- Context and Relevance are placeholders (not generated during conversion)
- `enriched_on` is not set by the script — it gets added by the `enrich-fathom-transcript` skill when enrichment completes (format: `YYYY-MM-DD HH:MM UTC`)

### Script Behavior

- **Deduplication**: skips meetings where the output file already exists (use `--force` to override)
- **Missing data**: uses graceful placeholders (`_No summary available._`, `No transcript captured.`, etc.)
- **Participant names**: uses speaker display names from transcript, falls back to calendar invitee names, then email local part
- **Null assignees**: action items without assignees show as "Unassigned"
- **Duration**: calculated from recording timestamps, falls back to scheduled times
- **No external dependencies**: Python 3 stdlib only

## Phase 2: LLM Enrichment (Optional)

After the script runs, generated files have placeholder Context and Relevance sections. To enrich them:

### Single File Enrichment

1. Read the generated `.md` file
2. Write the **Context** section (1-2 paragraphs):
   - Why this meeting happened
   - Where it fits in ongoing work
   - Relevant background from GrowthX context
3. Write the **Relevance** section (bullet groups):
   - Group by area: "To GrowthX Services:", "To CheckThat:", etc.
   - Include specific numbers, names, deadlines
4. Review and fix metadata if needed:
   - Participant name deduplication (e.g., "Ed" from email + "Edin Abazi" from transcript may be the same person)
   - Unidentified speakers (Fathom sometimes labels speakers as numbers like "2")

### Batch Enrichment

For large batches, process with Task agents in groups of 3-4. Each agent:
1. Reads the skeleton `.md` file
2. Generates Context and Relevance based on the transcript and summary content
3. Updates the file in place

### Writing Guidelines for Enrichment

- **Direct and clear** — no corporate jargon
- **Lead with the point** — most important info first
- **Use active voice** — "Marcel outlined" not "It was outlined by Marcel"
- **Ground in specifics** — numbers, names, dates, concrete details
- **Connect to company context** — reference GrowthX services, CheckThat, content strategy as relevant

## Post-Processing: Contact Dossiers & Index Files

After transcripts are saved, apply the **contact-transcript linking** rule (`.cursor/rules/contact-transcript-linking.mdc`):

1. **Identify** all named external participants
2. **Match** against existing dossiers in `records/contacts/`
3. **Update existing dossiers**: add transcript to Meeting Transcripts, add Timeline entry, bump `last_updated`
4. **Auto-create dossiers** for significant new contacts with `confidence: low`
5. **Tag each transcript** with a `## Related Contacts` section
6. **Update INDEX files**: `records/contacts/INDEX.md` and `records/transcripts/INDEX.md`

For large batches, run linking as a batch — collect all participants, deduplicate, then update contacts and indexes once.

## Workflow Summary

### Quick Convert (no enrichment)

User says: "Convert the fathom meetings from June"

1. Run the script: `python3 convert-to-transcript.py --month 2025-06`
2. Report results (written count, skipped count)
3. Done — LLM enrichment can happen later

### Full Convert with Enrichment

User says: "Convert and enrich the last 3 fathom meetings"

1. Run the script on specified meetings
2. Read each generated file
3. Generate Context and Relevance sections
4. Fix any metadata issues (participant names, unknown speakers)
5. Run contact-transcript linking
6. Update INDEX files
7. Report back with summary table

### Report Format

After processing, report:

| # | Date | File | Duration | Status |
|---|------|------|----------|--------|
| 1 | Jun 30 | `2025-06/2025-06-30-ed-jason-weekly.md` | 16m | wrote |
| 2 | Jun 28 | `2025-06/2025-06-28-team-standup.md` | 45m | skip (exists) |

Include: total written, total skipped, any meetings with no transcript or no summary.

## Edge Cases

- **No summary from Fathom**: older meetings (pre-2024-11 approx) may lack summaries. The script writes `_No summary available._` for all affected sections.
- **No action-items.json**: older meetings don't have this file. Falls back to "Next Steps" from summary, or `_No action items recorded._`.
- **No transcript**: if both `transcript.json` and `transcript-raw.json` are missing, writes `No transcript captured.`
- **Duplicate with Fireflies**: some meetings exist in both Fathom and Fireflies. The script skips if a file with the same name exists. Fireflies-sourced transcripts (with `fireflies_id` in metadata) take precedence.
