## Context

Outreach is experiencing critical data integrity issues with their Google Analytics 4 implementation on a Next.js headless application. The website architecture uses Google Tag Manager (GTM) loaded in the `<head>` of app.tsx for GA4 tracking, with OneTrust managing user consent and triggering GTM tags. Andrew discovered significant variance between actual user behavior and GA4 reported metrics (sessions, Google search clicks), which strongly correlates with UTM parameter duplication. This is an unusual problem for both participants despite their combined 26+ years of web analytics and tracking experience. The investigation requires technical code review and GTM configuration analysis to isolate whether the issue stems from the Next.js framework's page-loading behavior, the GTM implementation, the OneTrust consent trigger system, or UTM parameter construction in the codebase.
